QUICK START GUIDE
================================================================================

1. PUSH WORKFLOWS TO REPOSITORY
   $ git add .github/workflows/
   $ git add scripts/run-tests.sh scripts/docker-deploy.sh
   $ git commit -m "feat: Add comprehensive CI/CD infrastructure"
   $ git push origin main

2. VERIFY IN GITHUB
   - Go to repository Actions tab
   - Wait 2-3 minutes for workflows to appear
   - Watch workflows execute

3. CONFIGURE BRANCH PROTECTION (OPTIONAL)
   - Go to Settings > Branches > main
   - Enable "Require status checks to pass before merging"
   - Select: Build & Lint, Test Suite, Security Scanning, Docker Build

4. TEST LOCALLY
   $ ./scripts/run-tests.sh all        # Run all tests
   $ ./scripts/docker-deploy.sh        # Deploy locally
   $ npm run test:unit                 # Run unit tests only

5. MONITOR WORKFLOWS
   - Check Actions tab for workflow status
   - Review logs if failures occur
   - Check artifacts for test results
   - Monitor performance metrics

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

1. Workflow won't run
   → Check if .github/workflows/ is pushed
   → Verify branch protection rules
   → Check Actions are enabled in Settings

2. Tests failing locally but passing in CI
   → Use 'npm ci' instead of 'npm install'
   → Check Node.js version matches (20.x)
   → Verify no hardcoded paths

3. Docker build failing
   → Test locally: docker build .
   → Check Dockerfile syntax
   → Review build logs in Actions

4. Deployment health check timeout
   → Increase wait attempts in docker-deploy.sh
   → Check if app startup is slow
   → Review container logs

See .github/workflows/README.md Troubleshooting section for details.

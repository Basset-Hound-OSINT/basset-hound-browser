#### Blocked Components
- ✗ Apply fingerprint command fails: "No profile specified and no active profile"
- ✗ Configure canvas noise fails due to same root cause
- ✗ Get evasion config fails: Cannot retrieve configuration for profile

#### Root Cause: WebSocket Parameter Handling Bug

The WebSocket server has a parameter destructuring issue:

```javascript
// Current (incorrect) implementation
async handleCommand(data, options = {}) {
  const { command, ...params } = data;  // params = { params: {...} }
  // ...
  const result = await handler(params);  // Handler receives { params: {...} }
}

// When client sends: { command: 'xxx', params: { profileId: '...' } }
// Handler receives: { params: { profileId: '...' } }
// Handler looks for: params.profileId (undefined)
```

**Impact:** All commands that use named parameters (apply_fingerprint, configure_canvas_noise, set_evasion_levels, configure_webgl_noise, etc.) fail.

**Expected After Fix:**
- Canvas evasion at "aggressive" level: 82% effectiveness (from Phase 2 research)
- WebGL evasion at "aggressive" level: 90% effectiveness (from Phase 2 research)
- Audio fingerprinting evasion: 75-82% ceiling (hardware oscillator limitations)

#### Root Cause Analysis

Similar to the evasion system issue, behavioral commands fail due to parameter nesting:

```javascript
// Expected by handler
params.start = { x: 100, y: 100 }
params.end = { x: 500, y: 400 }

// Actual parameters received
params.params.start = { x: 100, y: 100 }  // Undefined error when accessing params.x
```

**Expected After Fix:**
- Mouse movement: 100-500 point paths with realistic Fitts Law timing
- Typing: Events with keystroke timing matching human behavior (50-100 WPM)
- Scrolling: Multi-event scroll patterns with momentum and pauses

## DETECTED ISSUES & SEVERITY

### CRITICAL (Blocks evasion deployment)

**Issue 1: WebSocket Parameter Destructuring Bug**
- **Severity:** CRITICAL
- **Scope:** Affects 12+ WebSocket commands
- **Location:** `/websocket/server.js` line 8284
- **Current Code:**
  ```javascript
  const { command, ...params } = data;
  // When data = { command: 'xxx', params: {...} }
  // Results in params = { params: {...} }
  ```
- **Fix:** Change to:
  ```javascript
  const { command, params } = data;
  // Or if params might be at top level for backward compat:
  const { command, ...rest } = data;
  const params = rest.params || rest;
  ```
- **Affected Commands:**
  - apply_fingerprint
  - configure_canvas_noise
  - configure_webgl_noise
  - configure_audio_noise
  - configure_font_evasion
  - set_evasion_levels
  - get_evasion_config
  - generate_mouse_path
  - generate_typing_events
  - set_active_fingerprint (returns false but sets active internally)

STANDARDS COMPLIANCE
--------------------

  - Support for external RFC 3161 TSA servers (configurable)

• ISO 27037:2012: Digital Evidence Identification and Preservation
  - Complete chain of custody from creation to disposal
  - Evidence integrity verification through hashing
  - Detailed metadata capture for each evidence item
  - Audit trail for all evidence access

• SWGDE: Scientific Working Group on Digital Evidence
  - Court-compliant forensic report generation
  - All SWGDE report writing requirements met
  - Case information, examiner identification, procedures

• NIST IR 8387: Digital Evidence Preservation
  - Cryptographic hashing (SHA-256) for integrity
  - Immutable evidence sealing
  - Comprehensive audit logging
  - Protection against unauthorized access

TEMPLATE STRUCTURE
------------------
{
  "name": "Template Name",
  "platform": "platform-name",
  "type": "profile | post | article | custom",
  "fields": {
    "fieldName": {
      "selector": "CSS selector",           // Single selector
      "selectors": ["sel1", "sel2"],        // Multiple fallback selectors
      "attribute": "textContent | href | src | content",
      "required": true | false,             // Field requirement
      "multiple": true | false,             // Extract array of values
      "transform": "extractNumber | toLowerCase | toUpperCase | trim"
    }
  }
}

Example:
{
  "name": "Custom Profile Template",
  "platform": "custom",
  "type": "profile",
  "fields": {
    "username": {
      "selector": ".username",
      "attribute": "textContent",
      "required": true,
      "transform": "trim"
    },
    "followers": {
      "selector": ".follower-count",
      "attribute": "textContent",
      "transform": "extractNumber"
    }
  }
}

SELECTOR TYPES
--------------
1. CSS Selectors (Primary)
   - Standard CSS selector syntax
   - querySelector and querySelectorAll support
   - Example: 'div.profile h1.name'

2. Multiple Selector Fallbacks
   - Try selectors in order until match found
   - Useful for site variations
   - Example: ['h1.title', 'meta[property="og:title"]']

3. Multiple Attribute Fallbacks
   - Try attributes in order
   - Example: 'content|textContent' tries content first, then textContent

4. XPath Support (Future)
   - Planned for future release
   - More powerful than CSS selectors

5. Regex Patterns (Future)
   - Pattern-based extraction
   - Text parsing and cleaning

TRANSFORM FUNCTIONS
-------------------
• extractNumber - Extract numeric value from string
  Example: "1,234 followers" → 1234

• toLowerCase - Convert to lowercase
  Example: "NAME" → "name"

• toUpperCase - Convert to uppercase
  Example: "name" → "NAME"

• trim - Remove leading/trailing whitespace
  Example: "  text  " → "text"

• Custom transforms - Add via template manager

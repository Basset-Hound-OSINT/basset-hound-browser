### Architecture Decisions

#### 1. Forensic Certification Path

**Architecture:** Modular compliance framework

```
src/forensics/
├── iso27037-validator.js         # Compliance checking
├── digital-signature.js           # PKI + signing
├── chain-of-custody.js            # SWGDE logging
├── audit-trail.js                 # External witness
└── compliance-report.js           # Audit documentation
```

**Key Design Decisions:**
- Cryptographic signing (RSA-2048 minimum, future: ECC-256)
- External timestamp service (NIST) for proof-of-time
- Modular validation (can check compliance without certification)
- Zero impact on existing functionality (isolated module)

---

#### 2. Competitor Monitoring Service

**Architecture:** Distributed scheduler + event-driven alerts

```
src/monitoring/
├── scheduler.js                  # Job orchestration
├── change-detector.js            # Diff engine
├── alert-dispatcher.js           # Multi-channel notifications
└── intelligence-report.js        # Report generation
```

**Key Design Decisions:**
- Cron-like scheduling (familiar for ops teams)
- Pluggable alert channels (email, Slack, webhook)
- Incremental change detection (only changes alert)
- Report generation can be daily, weekly, or on-demand

---

#### 3. Agent SDKs

**Architecture:** Lightweight wrappers around WebSocket API

```
sdk/
├── python/
│   ├── basset/
│   │   ├── client.py            # WebSocket wrapper
│   │   ├── types.py             # Type definitions
│   │   └── utils.py             # Helpers
│   └── examples/                # 20+ example scripts
└── javascript/
    ├── basset/
    │   ├── client.js            # WebSocket wrapper
    │   ├── types.d.ts           # TypeScript definitions
    │   └── utils.js             # Helpers
    └── examples/                # 20+ example scripts
```

**Key Design Decisions:**
- Minimal abstraction (direct 1:1 mapping to WebSocket API)
- Async/await first (modern Python/JavaScript patterns)
- Type-safe (TypeScript definitions, Python type hints)
- Examples-driven documentation (learn by example)

---

#### 4. Extended Session Support

**Architecture:** State machine + checkpoint recovery

```
src/session/
├── snapshot.js                  # State capture
├── recovery-orchestrator.js     # Recovery logic
├── failure-detector.js          # Issue classification
├── branching.js                 # Session trees
└── persistence.js              # Storage backend
```

**Key Design Decisions:**
- Checkpoint-based recovery (resume from known state)
- Automatic failure classification (know why session failed)
- A/B testing support (session branching for experiments)
- Graceful degradation (switch strategy if detection occurs)

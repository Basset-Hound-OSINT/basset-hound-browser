## Overview

This document provides step-by-step procedures for deploying v12.9.0 through a staged rollout approach, minimizing risk while validating the release at each stage.

**Deployment Timeline:**
- **Pre-Deployment:** 45 minutes
- **Canary Deployment:** 15 minutes (setup) + 4 hours (monitoring)
- **Beta Rollout:** 20 minutes (setup) + 2 hours (monitoring)
- **Production Rollout:** 30 minutes (full deployment)
- **Post-Deployment Validation:** 1 hour

**Total Deployment Window:** ~8.5 hours (distributed)

**Risk Level:** LOW (canary approach with automated rollback triggers)

## Pre-Deployment Checklist

**Duration:** 45 minutes  
**Responsible:** Deployment Lead + Technical Lead

### Phase 1: Code Verification (10 minutes)

```bash
# 1.1 Verify v12.9.0 branch/tag exists
git tag -l | grep v12.9.0
git branch -v | grep -E '(main|release)'

# 1.2 Confirm commit is ready
git log -1 --format="%H %s %ad" --date=short
# Should show v12.9.0 tag or release commit

# 1.3 Run full test suite
npm install
npm test

# BLOCKER: If test pass rate < 90%, do not proceed
# Expected: 320+/342 tests passing (93%+)
```

**Validation Checklist:**
- [ ] v12.9.0 tag/branch exists
- [ ] Latest commit shows v12.9.0 release
- [ ] `npm install` completes without errors
- [ ] Test suite runs to completion
- [ ] Pass rate ≥ 90% (≥320 tests)
- [ ] Zero CRITICAL test failures

### Phase 2: Code Quality & Security (10 minutes)

```bash
# 2.1 Lint and build
npm run lint
npm run build

# 2.2 Security audit
npm audit
npm audit fix  # Only if no breaking changes

# 2.3 Dependency check
npm ls 2>&1 | grep -i "missing\|invalid" || echo "Dependencies OK"

ls -lh dist/ | head -10

**Validation Checklist:**
- [ ] Linting passes (zero errors)

- [ ] Dependency tree valid (no missing/invalid packages)

Start Time: $(date)
Expected Duration: 4.5 hours (15 min setup + 4 hour monitoring)
Rollback Plan: Automatic if metrics exceed thresholds
Status Updates: #basset-deployments every 30 minutes
EOF

# Post to team
# (Send message to #basset-deployments)
```

### Step 2: Image Push to Canary Registry (3 minutes)

```bash
# 2.1 Push v12.9.0 image

docker push $REGISTRY/basset-hound-browser:v12.9.0

# 2.2 Verify push
curl -s $REGISTRY/v2/basset-hound-browser/tags/list | jq '.tags[] | select(. == "v12.9.0")'

# 2.3 Confirm image available on canary host
ssh basset-canary-prod "docker image pull $REGISTRY/basset-hound-browser:v12.9.0"
ssh basset-canary-prod "docker images | grep v12.9.0"
```

**Validation:**
- [ ] Image pushed to registry
- [ ] Pull successful on canary host
- [ ] Image digest matches local build

### Step 3: Canary Container Deployment (5 minutes)

```bash
# 3.1 SSH to canary host
ssh basset-canary-prod

# 3.2 Record baseline metrics (v12.8.0)
BASELINE_TIME=$(date +%s)
docker stats basset-hound-browser-prod --no-stream > /tmp/baseline-v12.8.0.txt
curl -s http://localhost:8765/stats > /tmp/stats-v12.8.0.json
echo "Baseline captured at: $BASELINE_TIME" > /tmp/baseline-marker.txt

# 3.3 Graceful shutdown of v12.8.0
docker stop basset-hound-browser-prod --time=30
sleep 5
docker rm basset-hound-browser-prod

# 3.4 Start v12.9.0 canary container
docker run -d \
  --name basset-hound-browser-prod \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e DISPLAY=:99 \
  -e ELECTRON_DISABLE_SANDBOX=1 \
  --cap-drop ALL \
  --cap-add SYS_ADMIN \
  --restart unless-stopped \
  --health-cmd='curl -f http://localhost:8765/health || exit 1' \
  --health-interval=30s \
  --health-timeout=10s \
  --health-retries=3 \
  -v basset-data:/app/data \
  -v basset-logs:/app/logs \
  registry.basset-prod.local/basset-hound-browser:v12.9.0

# 3.5 Wait for startup
sleep 30

# 3.6 Verify container running
docker ps | grep basset-hound-browser-prod
docker exec basset-hound-browser-prod curl -s http://localhost:8765/health
```

**Validation:**
- [ ] v12.8.0 container stopped cleanly
- [ ] v12.9.0 container started successfully
- [ ] Health endpoint responds
- [ ] Container health status shows "healthy"

### Step 4: Immediate Health Checks (2 minutes)

## Monitoring & Alerting

### Production Monitoring (First Hour)

**Every 5 minutes:**

```bash
# 5.1 Per-host health check
for i in 001 002 003; do
  HEALTH=$(curl -s -o /dev/null -w "%{http_code}" http://basset-prod-$i:8765)
  STATUS=$([ "$HEALTH" == "426" ] && echo "✓ OK" || echo "✗ FAIL")
  echo "basset-prod-$i: $STATUS (HTTP $HEALTH)"
done

# 5.2 Connection distribution
for i in 001 002 003; do
  CONNECTIONS=$(curl -s http://basset-prod-$i:8765/stats | jq '.activeConnections')
  echo "basset-prod-$i: $CONNECTIONS connections"
done

# 5.3 Error rate across cluster
for i in 001 002 003; do
  ERRORS=$(docker logs --since 5m basset-hound-browser-prod 2>&1 | grep -c ERROR || echo "0")
  echo "basset-prod-$i: $ERRORS errors in last 5 minutes"
done
```

### Automated Rollback Triggers

**Immediate rollback if ANY trigger fires:**

```bash
# T1: Multiple hosts down
DOWN_COUNT=$(for i in 001 002 003; do
  HEALTH=$(curl -s -o /dev/null -w "%{http_code}" http://basset-prod-$i:8765)
  [ "$HEALTH" != "426" ] && echo "1"
done | wc -l)

if [ $DOWN_COUNT -gt 1 ]; then
  echo "ROLLBACK TRIGGER: 2+ hosts down"
  # Execute rollback (see section below)
fi

# T2: Error rate > 5%
ERROR_RATE=$(curl -s http://basset-loadbalancer-prod/metrics | grep basset_error_rate | awk '{print $2}' | sed 's/%//')
if (( $(echo "$ERROR_RATE > 5" | bc -l) )); then
  echo "ROLLBACK TRIGGER: Error rate > 5%"
  # Execute rollback
fi

# T3: Memory spike on all hosts
for i in 001 002 003; do
  MEMORY=$(ssh basset-prod-$i "docker stats basset-hound-browser-prod --no-stream" | awk '{print $7}' | sed 's/[^0-9.]//g')
  if (( $(echo "$MEMORY > 750" | bc -l) )); then
    echo "ROLLBACK TRIGGER: Host $i memory > 750MB"
    # Execute rollback
  fi
done

# T4: Latency degradation
P95_LATENCY=$(curl -s http://prometheus:9090/api/v1/query?query=basset_latency_p95 | jq '.data.result[0].value[1]')
if (( $(echo "$P95_LATENCY > 500" | bc -l) )); then  # 500ms threshold
  echo "ROLLBACK TRIGGER: Latency P95 > 500ms"
  # Execute rollback
fi
```

## Rollback Procedures

### Quick Rollback (Automatic - On Trigger)

**When:** Any automatic rollback trigger fires  
**Duration:** 10 minutes per host  

```bash
#!/bin/bash
# Automatic rollback script

for HOST in basset-prod-001 basset-prod-002 basset-prod-003; do
  echo "Rolling back $HOST to v12.8.0..."
  
  ssh $HOST << 'EOF'
  # 1. Stop v12.9.0
  docker stop basset-hound-browser-prod --time=10
  docker rm basset-hound-browser-prod
  
  # 2. Start v12.8.0 (pre-deployment rollback image)
  docker run -d \
    --name basset-hound-browser-prod \
    --network basset-hound-browser \
    -p 8765:8765 \
    --restart unless-stopped \
    -v basset-data:/app/data \
    -v basset-logs:/app/logs \
    basset-hound-browser:pre-v12.9.0-rollback
  
  sleep 30
  
  # 3. Verify
  HEALTH=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8765)
  if [ "$HEALTH" == "426" ]; then
    echo "✓ Rollback successful on $HOST"
  else
    echo "✗ Rollback verification failed on $HOST"
    exit 1
  fi
EOF
done

# 4. Announce rollback
echo "[ROLLBACK] v12.9.0 rollback to v12.8.0 COMPLETE"
echo "Trigger: [SPECIFY TRIGGER]"
echo "Time: $(date)"
# Post to #basset-deployments
```

### Manual Rollback (If Automatic Fails)

**When:** Automatic rollback fails or manual intervention needed  

```bash
# Manual step-by-step rollback:

# Step 1: Drain all traffic to v12.8.0 stable instances
ssh basset-loadbalancer-prod << 'EOF'
# Remove v12.9.0 hosts from upstream
cat > /etc/nginx/conf.d/basset-hound.conf << 'LB'
upstream basset_production {
  server basset-prod-rollback-stable:8765;
}
LB
systemctl reload nginx
EOF

# Step 2: Restore data from backup
for i in 001 002 003; do
  ssh basset-prod-$i << 'EOF'
  docker stop basset-hound-browser-prod
  docker rm basset-hound-browser-prod
  
  # Restore data from backup
  docker cp /backups/data-v12.8.0-prod-$i/. basset-hound-browser-prod:/app/data
EOF
done

# Step 3: Start v12.8.0 containers
for i in 001 002 003; do
  ssh basset-prod-$i << 'EOF'
  docker run -d \
    --name basset-hound-browser-prod \
    --network basset-hound-browser \
    -p 8765:8765 \
    -v basset-data:/app/data \
    basset-hound-browser:v12.8.0
  
  sleep 20
  docker exec basset-hound-browser-prod curl -s http://localhost:8765/health
EOF
done

# Step 4: Restore normal traffic distribution
ssh basset-loadbalancer-prod << 'EOF'
cat > /etc/nginx/conf.d/basset-hound.conf << 'LB'
upstream basset_production {
  server basset-prod-001:8765 weight=1;
  server basset-prod-002:8765 weight=1;
  server basset-prod-003:8765 weight=1;
}
LB
systemctl reload nginx
EOF
```

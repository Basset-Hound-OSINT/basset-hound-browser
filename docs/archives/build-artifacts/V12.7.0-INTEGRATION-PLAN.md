### FEATURE 3: Extended Evasion Vectors Commands

**File:** `/websocket/commands/extended-evasion-commands.js`

**Commands to Register:**

1. **`enable_tls_evasion`** (60 lines)
   - Input: `cipherSuites` (array), `tlsVersions` (array), `rotateOnEachRequest` (boolean)
   - Output: `enabled`, `activeCiphers`, `tlsVersions`
   - Action: Configure TLS fingerprinting evasion
   - Uses: Proxy manager, request interceptor
   - Error cases: Invalid cipher, TLS version not supported

2. **`enable_http2_evasion`** (55 lines)
   - Input: `randomizeStreamPriority` (boolean), `manipulateHeaders` (boolean), `varyFlowControl` (boolean)
   - Output: `enabled`, `techniques[]`
   - Action: Enable HTTP/2 specific evasion techniques
   - Uses: Request interceptor, network analysis manager
   - Error cases: HTTP/2 not available, config conflict

3. **`enable_timing_evasion`** (50 lines)
   - Input: `randomization` ('low'|'medium'|'high'), `jitterRange` (ms, default 20-50)
   - Output: `enabled`, `jitterMs`, `randomizationLevel`
   - Action: Enable request timing randomization
   - Uses: Timing coordinator (new or existing)
   - Error cases: Invalid jitter range

4. **`set_evasion_profile`** (45 lines)
   - Input: `profile` ('minimal'|'balanced'|'aggressive'), `overrides` (optional)
   - Output: `active`, `techniques[]`, `estimatedDetectionRate`
   - Action: Apply pre-configured evasion profile
   - Profiles:
     - minimal: TLS + basic timing
     - balanced: TLS + HTTP2 + timing (recommended)
     - aggressive: All techniques + behavioral patterns
   - Error cases: Invalid profile

5. **`get_evasion_status`** (35 lines)
   - Input: None
   - Output: `activeTechniques[]`, `tlsConfig`, `timingConfig`, `behavioralConfig`, `detectionRate`
   - Action: Query current evasion state
   - Error cases: None

6. **`test_evasion_effectiveness`** (60 lines)
   - Input: `targetService` ('perimetrix'|'datadome'|'cloudflare'|'recaptcha'|'distil'), `testCount` (3-10)
   - Output: `results[]` with service, evasionScore (0-100), detectionAttempts, successRate
   - Action: Run detection tests, return scores
   - Uses: Detection service probes (requires test infrastructure)
   - Error cases: Service unreachable, timeout

```javascript
function registerMonitoringMetricsCommands(server, mainWindow) {
  const { registerMonitoringHandlers } = require('../../src/monitoring/websocket-handlers');
  
  // Initialize metrics stack if not already done
  if (!server.metricsCollector) {
    const { MetricsCollector } = require('../../src/monitoring/metrics-collector');
    const { MetricsAggregator } = require('../../src/monitoring/metrics-aggregator');
    const { AlertManager } = require('../../src/monitoring/alert-manager');
    const { MetricsStore } = require('../../src/monitoring/metrics-store');
    
    server.metricsCollector = new MetricsCollector();
    server.metricsAggregator = new MetricsAggregator();
    server.alertManager = new AlertManager();
    server.metricsStore = new MetricsStore();
  }
  
  registerMonitoringHandlers(
    server.commandDispatcher,
    server.metricsCollector,
    server.metricsAggregator,
    server.alertManager,
    server.metricsStore,
    server
  );
}
```

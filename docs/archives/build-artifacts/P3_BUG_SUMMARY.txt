P3-001: SCREENSHOT MEMORY LEAKS (2 hours)
────────────────────────────────────────
File: src/extraction/screenshot-phase4-robustness.js
Root Cause: Recovery logs accumulate unbounded, stream handles leak, error
            reports retain full context objects
Impact: Memory pressure over time during long-running screenshot operations
Test Cases: 10 (CircularBuffer bounds, stream cleanup, pooling, growth limits)
Fix Complexity: Medium
Key Changes:
  • Replace array-based logging with CircularBuffer (bounded O(n) to O(1))
  • Add stream cleanup via setImmediate after compression fallback
  • Implement object pool for error reports (max 50 cached objects)
  • Trigger GC when memory pressure > 100MB
  • Add try-finally cleanup handlers

Success Metric: Memory delta < 5MB over 10,000 screenshot operations

────────────────────────────────────────────────────────────────────────────────

P3-002: SESSION COHERENCE EDGE CASES (2 hours)
──────────────────────────────────────────────
File: src/evasion/session-coherence.js
Root Cause: False positives on legitimate state changes (fingerprint updates,
            screen rotation, network patterns), race conditions in concurrent
            state updates, outdated impossible device combinations
Impact: Legitimate user sessions flagged as suspicious; evasion framework
        incorrectly rejects valid scenarios
Test Cases: 10 (exemptions, rotation, variance, Chrome OS, race conditions)
Fix Complexity: Medium-High
Key Changes:
  • Add allowedComponentUpdates configuration (fonts, screen, etc.)
  • Skip exempt components in temporal validation layer
  • Replace variance threshold with coefficient of variation (human-like 0.15+)
  • Implement screen orientation normalization (sort dimensions)
  • Update impossible device combinations (Chrome OS + Chrome valid, iPad + Firefox valid)
  • Add atomic locking to recordInteraction for race condition prevention

Success Metric: Coherence score > 0.90 for legitimate sessions with valid changes

────────────────────────────────────────────────────────────────────────────────

P3-003: TIMEOUT HANDLER CLEANUP FAILURES (2 hours)
──────────────────────────────────────────────────
File: src/resilience/timeout-protection.js
Root Cause: Timeout handlers created but never cancelled if promises never
            settle; activeTasks accumulate without cleanup; retry logic
            doesn't cancel prior timeout handlers; missing emergency cleanup
Impact: Under high concurrency (1000+), handlers accumulate → memory pressure
        → eventual system hang/OOM
Test Cases: 10 (cleanup on resolution, activeTasks drainage, retry cleanup,
                overdue detection, 1000+ concurrent)
Fix Complexity: High
Key Changes:
  • Implement CancellableTimeout class with AbortController
  • Rewrite withTimeout() to use AbortController + guaranteed finally cleanup
  • Add activeTimeouts tracking Map
  • Implement cleanup watcher (30-sec interval) to detect/cleanup overdue tasks
  • Update executeWithFallback retry loop to cancel prior timeouts
  • Enhance trackTask with guaranteed cleanup via wrapped promise
  • Add destroy() method for process shutdown cleanup

Success Metric: activeTasks Map always empty after timeouts resolve;
               no dangling setTimeout/setInterval

────────────────────────────────────────────────────────────────────────────────

P3-004: ERROR LOGGING CONTEXT LOSS (2-3 hours)
──────────────────────────────────────────────
File: src/observability/error-tracer.js
Root Cause: Unbounded context storage (10MB+ DOM trees stored as-is);
            unindexed error search (O(n), becomes O(n²) in loops);
            unbounded pattern collections; unlimited recovery attempts;
            full stack traces retained with closure variables
Impact: Large context objects cause heap growth (GB scale); error queries
        timeout; difficult to debug large error sets
Test Cases: 10 (context validation, circular refs, search performance,
                collection bounds, recovery attempt limits)
Fix Complexity: High
Key Changes:
  • Add context size validation (MAX_CONTEXT_SIZE = 1MB)
  • Implement JSON serialization with circular reference detection
  • Add error indexing (byType, byComponent, bySeverity, bySpan Maps)
  • Rewrite findRelatedErrors() to use index lookup (O(1) vs O(n))
  • Bound recovery attempt array to 10 max (FIFO eviction)
  • Limit error pattern Sets to 100 entries each
  • Limit stack trace depth to 15 frames
  • Enhance close() cleanup to deallocate all indexes

Success Metric: Error search < 10ms for 10K errors; memory < 10MB for
               10K errors with full context

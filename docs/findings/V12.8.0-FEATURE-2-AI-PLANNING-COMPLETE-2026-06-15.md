## ARCHITECTURE OVERVIEW

### Three Core Capabilities

#### 1. Autonomous Task Execution (600-900 LOC, 35 tests)

Transform natural language into executable browser operations:

```
"Log into GitHub and find basset-hound-browser repo"
         ↓ (Natural Language Parser)
{ intent: "login_and_search", steps: [...], dependencies: [...] }
         ↓ (Task Orchestrator)
execute navigate → fill form → submit → wait → search
         ↓ (Result Aggregator)
{ success: true, authenticated: true, results: [...], timing: 12.4s }
```

**Modules:**
- Natural Language Parser (150-200 LOC, 10 tests): Intent extraction, parameter recognition
- Task Orchestrator (250-350 LOC, 15 tests): Execution planning, dependency management
- Result Aggregator (200-300 LOC, 10 tests): Format for AI consumption

#### 2. Adaptive Evasion (500-800 LOC, 30 tests)

Real-time detection analysis and tactical adjustment:

```
Cloudflare Challenge Detected
         ↓ (Detection Analyzer)
{ type: "js_challenge", confidence: 0.95, indicators: [...] }
         ↓ (Recommendation Engine)
{ tactics: ["header_rotation", "timing_injection"], effectiveness: 0.72 }
         ↓ (Real-time Adjuster)
apply tactics → measure → success → continue
```

**Modules:**
- Detection Analyzer (150-250 LOC, 10 tests): Identify challenge type
- Recommendation Engine (200-300 LOC, 10 tests): Suggest tactics
- Real-time Adjuster (150-250 LOC, 10 tests): Apply and verify

#### 3. Cross-Domain Context (400-600 LOC, 25 tests)

Maintain coherent AI state across multiple domains:

```
Domain A: GitHub
  - authenticated: true
  - user: test@example.com
  - context: [repos, profile data]
         ↓ (Context Manager)
Domain B: LinkedIn
  - authenticated: true
  - user: same person (verified)
  - context: [profile, connections]
         ↓ (Multi-Domain Orchestrator)
Unified AI view → coordinated operations
```

**Modules:**
- Session Context Manager (150-250 LOC, 10 tests): Store/retrieve domain state
- Multi-Domain Orchestrator (150-250 LOC, 10 tests): Coordinate across domains
- AI Integration Points (100-150 LOC, 5 tests): Claude API, palletai agents

## NEW WEBSOCKET COMMANDS (15 total)

### Autonomous Task Execution (4 commands)

```javascript
// Execute natural language directive
ws.send({ id: 1, command: "execute_task", 
  directive: "Log into GitHub and find basset-hound-browser",
  timeout: 60000 });

// Response:
{ id: 1, success: true, steps_completed: 5, execution_time: 12450,
  results: { authenticated: true, repos: [...] } }
```

**Commands:**
- `execute_task` - Run natural language directive
- `parse_task` - Parse without executing
- `get_task_progress` - Check progress of running task
- `cancel_task` - Stop running task

### Adaptive Evasion (5 commands)

```javascript
// Analyze response for detection
ws.send({ id: 2, command: "analyze_detection",
  response_data: { headers: {...}, content: "...", status_code: 200 } });

// Response:
{ id: 2, success: true, detected: true, type: "cloudflare_challenge",
  confidence: 0.95, indicators: [...] }
```

**Commands:**
- `analyze_detection` - Detect challenges in response
- `recommend_evasion` - Get tactic recommendations
- `apply_evasion` - Apply recommended tactics
- `measure_evasion_effectiveness` - Verify tactic success
- `get_evasion_effectiveness_stats` - View tactic history

### Cross-Domain Context (6 commands)

```javascript
// Store domain context
ws.send({ id: 3, command: "store_context",
  domain: "github.com", context: { authenticated: true, user: "test@example.com" } });

// Response:
{ id: 3, success: true, context_id: "ctx_abc123" }
```

**Commands:**
- `store_context` - Save domain-specific state
- `get_context` - Retrieve domain context
- `link_contexts` - Link contexts across domains
- `execute_multi_domain` - Execute across domains
- `get_conversation_history` - View interaction history
- `recover_context` - Restore from disconnect

## RISK MITIGATION

### High-Priority Risks

**Risk:** NLP parsing ambiguity  
**Probability:** Medium | **Impact:** High  
**Mitigation:** 
- Implement recovery suggestions flow
- Fallback to explicit command mode
- User clarification prompts

**Risk:** Detection evasion varies by site  
**Probability:** High | **Impact:** Medium  
**Mitigation:**
- Build site-specific learning system
- Track effectiveness per tactic/site combination
- Implement escalation tactics

**Risk:** Cross-domain context isolation  
**Probability:** Low | **Impact:** High  
**Mitigation:**
- Strict domain boundary enforcement
- Comprehensive validation checks
- Manual override capability

### Medium-Priority Risks

**Risk:** AI integration latency  
**Probability:** Medium | **Impact:** Low  
**Mitigation:**
- Async processing
- Response caching
- Batch processing mode

**Risk:** Memory usage with multi-domain contexts  
**Probability:** Low | **Impact:** Medium  
**Mitigation:**
- Context compression
- LRU cache eviction
- Storage optimization

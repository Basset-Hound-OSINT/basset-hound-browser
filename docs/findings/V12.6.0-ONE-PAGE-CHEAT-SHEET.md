## RED FLAGS (Stop & Debug)

🚩 **Real data not captured** (getting mocks)  
→ Data fetching broken, BLOCKER, debug immediately

🚩 **>50% site failures** (pattern indicates root cause)  
→ Not random issues, something fundamentally wrong, debug day 1

🚩 **Evasion 0% effective** (detection working perfectly)  
→ Evasion framework broken or detection stronger than expected

🚩 **Timeline slipping >2 days** (falling behind early)  
→ Scope too large, reduce or ask for extension

🚩 **0 bugs found in Phase 1** (too good to be true)  
→ Test coverage insufficient, dig deeper

## FAILURE MODE & RECOVERY

**IF Phase 1 fails (< 80% success):**  
- Extend Phase 1: 2-3 extra days for debugging
- Shift timeline: All subsequent phases slip 2-3 days
- New release date: July 10-12 (not July 8)
- Decision: Fix critical issues before release

**IF Phase 2 fails (< 95% test pass):**  
- Extend Phase 2: More bug fixes
- Shift timeline: 2-3 days
- New release date: July 10-12
- Decision: More testing, improve quality

**IF evasion tests show < 70% (Phase 4):**  
- Document limitations honestly
- Plan evasion improvements for v12.7.0
- Release v12.6.0 with known limitations
- Message: "Real measurements, not claims"

INTEGRATION GUIDE
================================================================================

Phase 2 modules are designed for easy integration into the existing detection
system. Each module is independent and can be used individually or together.

Usage Example - Full Detection Workflow:

  const {
    TechnologyDetectionEngine,
    VersionFingerprinter,
    VulnerabilityDetector,
    ConfigurationAnalyzer,
    UpdateRecommender
  } = require('./src/detection');

  // 1. Detect technologies (Phase 1)
  const detector = new TechnologyDetectionEngine();
  const detection = detector.detect({
    html: pageHTML,
    headers: responseHeaders
  });

  // 2. Get detailed version information (Phase 2)
  const fingerprinter = new VersionFingerprinter();
  const versionDetails = detection.technologies.map(tech => ({
    ...tech,
    versionDetails: fingerprinter.fingerprint(
      tech.name,
      tech.detection,
      tech.version,
      { html: pageHTML, headers: responseHeaders }
    )
  }));

  // 3. Check for vulnerabilities (Phase 2)
  const vulnDetector = new VulnerabilityDetector();
  const vulnAnalysis = versionDetails.map(tech => ({
    ...tech,
    vulnerabilities: vulnDetector.detectVulnerabilities(
      tech.name,
      tech.versionDetails.version
    )
  }));

  // 4. Analyze configuration (Phase 2)
  const analyzer = new ConfigurationAnalyzer();
  const configIssues = analyzer.analyzeConfiguration(
    { html: pageHTML, headers: responseHeaders },
    detection.technologies
  );

  // 5. Get upgrade recommendations (Phase 2)
  const recommender = new UpdateRecommender();
  const upgrades = versionDetails.map(tech => ({
    ...tech,
    upgrade: recommender.getRecommendation(
      tech.name,
      tech.versionDetails.version,
      'production'
    )
  }));

  // 6. Generate comprehensive report
  const report = {
    detection,
    versionDetails,
    vulnerabilities: vulnAnalysis,
    configurationIssues: configIssues,
    upgradeRecommendations: upgrades,
    summary: {
      totalTechnologies: detection.technologies.length,
      criticalVulnerabilities: vulnAnalysis.reduce(
        (sum, t) => sum + (t.vulnerabilities?.criticalVulnerabilities || 0),
        0
      ),
      configurationIssues: configIssues.totalIssues,
      recommendedUpgrades: upgrades.filter(t => t.upgrade.needsUpgrade).length
    }
  };

Integration Points:
  • Can extend existing TechnologyDetectionEngine
  • Works with current WebSocket API
  • Compatible with MCP server
  • Supports async operations
  • Error handling via try-catch blocks

Module Independence:
  • Version Fingerprinter: Standalone, no dependencies
  • Vulnerability Detector: Standalone, uses local CVE DB
  • Configuration Analyzer: Standalone, no external calls
  • Update Recommender: Standalone, uses local version DB
  • All modules: No external API calls (local data only)

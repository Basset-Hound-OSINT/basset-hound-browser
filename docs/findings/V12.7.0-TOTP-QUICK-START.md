## IMPLEMENTATION ARCHITECTURE

### Core Algorithm Implementation
- **TOTP:** RFC 6238 compliant with SHA-1/256/512 support
- **HOTP:** RFC 4226 compliant with counter tracking
- **Base32 Decoding:** Standard alphabet (A-Z, 2-7)
- **Time Window:** 30s and 60s support with drift tolerance (±1-2 windows)

### QR Code Parsing
- Extract QR code from registration screenshots
- Decode otpauth:// URI format
- Parse: `secret`, `issuer`, `account`, `algorithm`, `digits`, `period`
- Support manual URI input as fallback

### Secure Credential Storage
- **At Rest:** AES-256-GCM encryption
- **In Memory:** Zeroized after use
- **Session State:** Encrypted profile storage
- **Recovery:** Auto-import on session restore

### MFA Automation Workflows
1. **Registration:** QR detection → secret parsing → verification → storage
2. **Login:** Credential retrieval → token generation → form filling → submission
3. **Recovery:** Backup code detection → HOTP mode → counter management

### Provider Support (Tested)
✓ Google Authenticator  
✓ Microsoft Authenticator  
✓ GitHub 2FA  
✓ Authy  
✓ AWS MFA & Generic TOTP/HOTP

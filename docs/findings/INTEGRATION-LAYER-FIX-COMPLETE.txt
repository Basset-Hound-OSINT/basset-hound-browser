Issue #2: RESPONSE FORMAT MISMATCH
-----------------------------------
Problem: Integration tests expect responses wrapped in a `result` property, but handlers
return flat response objects. This causes "Cannot read properties of undefined" errors
when tests try to access response.result.property.

Pattern:
- Handler returns: { success: true, profile: {...} }
- Test expects:   response.result.profile
- Result:         response.result is undefined -> TypeError

TECHNICAL EXPLANATION
====================

The WebSocket integration harness works as follows:

1. MockExtension/MockBrowser sends command to TestServer:
   { id: "msg-123", command: "create_profile", ...params }

2. TestServer processes command via registered handler:
   const response = await this.commandHandlers.get(type)(params);

3. TestServer sends back response to client:
   { id: "msg-123", command: type, ...response }

   Note: Response is SPREAD into the final message

4. MockExtension resolves pending command with full message:
   resolve({ id: "msg-123", command: type, ...response })

Therefore, if handler returns { success: true, profile: {...} }, the client receives:
   { id: "msg-123", command: "create_profile", success: true, profile: {...} }

But if tests expect response.result.profile, the handler must return:
   { success: true, result: { profile: {...} } }

Which results in client receiving:
   { id: "msg-123", command: "create_profile", success: true, result: { profile: {...} } }

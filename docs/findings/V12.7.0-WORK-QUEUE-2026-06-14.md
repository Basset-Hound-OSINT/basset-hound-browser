## QUICK REFERENCE: FAILURE MODES

**If Feature X test failures >5:**
- Identify root cause
- Fix in working branch
- Re-run tests
- Reassess gate criteria
- Decision: Fix & continue OR defer feature to v12.8.0

**If performance regression >5%:**
- Profile to identify bottleneck
- Optimize critical path
- Re-measure
- Decision: Release v12.7.0 with note OR defer features

**If integration issues discovered:**
- Document conflict
- Apply mitigation from architecture doc
- Add integration test
- Retest all affected features
- Decision: Release v12.7.0 OR extend

**If docker build fails:**
- Identify compilation/dependency issue
- Fix in codebase
- Rebuild
- Decision: Release OR extend

**If at any point <95% tests passing:**
- GATE FAILS automatically
- Fix all failures
- Reassess timeline
- Extend release as needed

## APPENDIX: TEST TEMPLATES

### Unit Test Template
```javascript
describe('Feature X', () => {
  test('should do X when Y', () => {
    const result = functionUnderTest(input);
    expect(result).toBe(expected);
  });
});
```

### Integration Test Template
```javascript
describe('Feature X Integration', () => {
  let server;
  
  beforeAll(() => {
    server = startWebSocketServer();
  });
  
  afterAll(() => {
    server.close();
  });
  
  test('should handle command X', async () => {
    const result = await sendCommand(server, { ... });
    expect(result).toMatchObject({ ... });
  });
});
```

### Performance Test Template
```javascript
describe('Feature X Performance', () => {
  test('should complete in <100ms', async () => {
    const start = Date.now();
    const result = await heavyOperation();
    const duration = Date.now() - start;
    expect(duration).toBeLessThan(100);
  });
});
```

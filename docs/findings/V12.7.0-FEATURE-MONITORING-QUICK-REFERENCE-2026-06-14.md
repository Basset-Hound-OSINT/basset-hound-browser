## KEY IMPLEMENTATION NOTES

### 1. Latency Percentile Calculation
Use sorted array + index math (not full sort each time):
```javascript
function percentile(sorted, p) {
  const idx = Math.ceil((p / 100) * sorted.length) - 1;
  return sorted[idx];
}
```

### 2. Ring Buffer for Historical Storage
Use fixed-size circular buffers to avoid unbounded memory growth:
```javascript
class RingBuffer {
  constructor(size) { this.buffer = new Array(size); this.head = 0; }
  push(item) { this.buffer[this.head % this.size] = item; this.head++; }
}
```

### 3. Async Metrics Collection
Use setImmediate() for non-blocking metric recording:
```javascript
recordCommand(name, duration, success, bytes) {
  setImmediate(() => {
    // Update metrics without blocking command execution
  });
}
```

### 4. Alert Suppression
Track suppression by type with auto-expiry:
```javascript
const suppressions = new Map();
suppressions.set(alertType, Date.now() + durationMs);
// Cleanup:
for (const [type, until] of suppressions) {
  if (Date.now() > until) suppressions.delete(type);
}
```

### 5. Time-Window Boundaries
Handle boundary conditions for aggregation:
```javascript
function getWindowBucket(timestamp, windowMs) {
  return Math.floor(timestamp / windowMs) * windowMs;
}
// All events in 60s → same bucket
```

## COMMAND HANDLER TEMPLATE

```javascript
handlers['get_metrics'] = (ws, message, client) => {
  try {
    const metrics = metricsCollector.getCurrentMetrics();
    
    // Validate metrics
    if (!metrics || !metrics.timestamp) {
      throw new Error('Metrics not ready');
    }
    
    sendSuccess(ws, message.id, metrics);
  } catch (error) {
    sendError(ws, message.id, error.message);
  }
};

// For streaming:
handlers['stream_metrics'] = (ws, message, client) => {
  const { interval = 1000 } = message;
  const streamId = crypto.randomUUID();
  
  const intervalHandle = setInterval(() => {
    const metrics = metricsCollector.getCurrentMetrics();
    ws.send(JSON.stringify({
      type: 'metrics_update',
      streamId,
      data: metrics
    }));
  }, interval);
  
  // Track for cleanup
  client.streams = client.streams || new Map();
  client.streams.set(streamId, intervalHandle);
  
  sendSuccess(ws, message.id, { streamId });
  
  // Cleanup on disconnect
  ws.on('close', () => {
    clearInterval(intervalHandle);
    client.streams?.delete(streamId);
  });
};
```

## COMMON PITFALLS

1. **Not cleaning up old samples** → Memory grows unbounded
   - Use RingBuffer or explicit cleanup

2. **Blocking metric collection** → Slows down commands
   - Use setImmediate() or event emission

3. **Percentile calculation on unsorted data** → Wrong results
   - Always sort before calculating

4. **Alert spam** → Alerts trigger repeatedly
   - Implement suppression/deduplication

5. **Memory leaks in streaming** → Connections not cleaned up
   - Track stream IDs and cleanup on disconnect

6. **Time-window boundaries** → Off-by-one errors
   - Use floor() for consistent bucketing

7. **Unbounded alert history** → Memory growth
   - Keep fixed-size buffer (e.g., last 1000 alerts)

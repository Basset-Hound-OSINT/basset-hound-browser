## 1. TOTP GENERATOR MODULE

### File: `src/credentials/totp-generator.js`

#### RFC 6238 Compliance Requirements
TOTP (Time-based One-Time Password) is defined in RFC 6238 and is the standard algorithm used by Google Authenticator, Microsoft Authenticator, Authy, and most 2FA services.

#### Implementation Specification

```javascript
class TOTPGenerator {
  /**
   * Initialize TOTP generator with secret
   * 
   * @param {string} secret - Base32-encoded secret (from QR code or backup codes)
   * @param {Object} options - Configuration options
   *   - algorithm: 'SHA1' | 'SHA256' | 'SHA512' (default: 'SHA1')
   *   - window: 30 | 60 (seconds, default: 30)
   *   - digits: 6 | 7 | 8 (token length, default: 6)
   *   - epoch: number (default: 0, UNIX epoch start)
   */
  constructor(secret, options = {})
  
  /**
   * Generate current TOTP token
   * @returns {Object} { token: string, expiresAt: number, validFor: number }
   */
  generate()
  
  /**
   * Generate TOTP for specific timestamp
   * @param {number} timestamp - UNIX timestamp in milliseconds
   * @returns {Object} { token: string, window: number, counter: number }
   */
  generateAtTime(timestamp)
  
  /**
   * Validate a TOTP token (with drift tolerance)
   * @param {string} token - Token to validate
   * @param {number} window - Number of windows before/after current (default: 1)
   * @returns {boolean} true if token is valid
   */
  validate(token, window = 1)
  
  /**
   * Get time remaining before token expires
   * @returns {number} milliseconds until expiry
   */
  getTimeRemaining()
  
  /**
   * Get next token (useful for preemptive fill)
   * @returns {Object} { token: string, startsAt: number }
   */
  getNextToken()
  
  /**
   * Calculate current time counter
   * @returns {number} HMAC counter value
   */
  getCounter()
}
```

#### Key Implementation Details

1. **Base32 Secret Parsing**
   - Decode Base32 to binary secret (standard alphabet: A-Z, 2-7)
   - Handle padding correctly (standard RFC 4648)
   - Support secrets with/without padding
   - Example: `"JBSWY3DPEBLW64TMMQ===="`

2. **HMAC-Based Counter Generation**
   ```
   1. Obtain current time T (seconds)
   2. Calculate counter C = (T - epoch) / window
   3. Generate HMAC-SHA1(secret, counter)
   4. Extract 31-bit value from HMAC (dynamic truncation per RFC 6238)
   5. Modulo 10^digits to get final token
   ```

3. **Algorithm Support**
   - SHA-1 (default, compatibility with all services)
   - SHA-256 (for enhanced security)
   - SHA-512 (for maximum entropy)

4. **Time Window Support**
   - 30 seconds (most common, Google/Microsoft/GitHub/Authy)
   - 60 seconds (some enterprise systems)
   - Custom windows via options

5. **Drift Tolerance**
   - Default: ±1 window (±30 seconds at 30s window)
   - Configurable for out-of-sync recovery
   - Prevents rejection of valid tokens due to clock skew

6. **Token Expiry Calculation**
   - Calculate milliseconds until current token expires
   - Enable preemptive form fill before expiry
   - Track "time remaining" for UI display

## 2. HOTP GENERATOR MODULE

### File: `src/credentials/hotp-generator.js`

#### RFC 4226 Compliance Requirements
HOTP (HMAC-based One-Time Password) uses counter-based generation instead of time. Used for hardware tokens and backup codes.

#### Implementation Specification

```javascript
class HOTPGenerator {
  /**
   * Initialize HOTP generator with secret
   * 
   * @param {string} secret - Base32-encoded secret
   * @param {Object} options - Configuration options
   *   - algorithm: 'SHA1' | 'SHA256' | 'SHA512' (default: 'SHA1')
   *   - digits: 6 | 7 | 8 (token length, default: 6)
   *   - initialCounter: number (default: 0)
   */
  constructor(secret, options = {})
  
  /**
   * Generate token for next counter value
   * @returns {Object} { token: string, counter: number }
   */
  generate()
  
  /**
   * Generate token for specific counter value
   * @param {number} counter - Counter value
   * @returns {Object} { token: string, counter: number }
   */
  generateFor(counter)
  
  /**
   * Validate token with resync support
   * @param {string} token - Token to validate
   * @param {number} lookahead - Number of counters ahead to check (default: 0)
   * @returns {Object} { valid: boolean, counter: number }
   */
  validate(token, lookahead = 0)
  
  /**
   * Resynchronize with server counter
   * @param {number} correctCounter - Correct counter value from server
   */
  resync(correctCounter)
  
  /**
   * Get current counter
   * @returns {number}
   */
  getCounter()
  
  /**
   * Increment and get next counter
   * @returns {number}
   */
  incrementCounter()
}
```

#### Key Implementation Details

1. **Counter-Based Generation**
   ```
   1. Counter C (maintained by client and server)
   2. Generate HMAC-SHA1(secret, counter)
   3. Extract 31-bit value (dynamic truncation)
   4. Modulo 10^digits to get token
   ```

2. **Resynchronization**
   - Support out-of-sync tokens (user pressed button multiple times)
   - Lookahead window to find matching token
   - Prevent counter rollback (security)

3. **Counter Persistence**
   - Store counter state in credential manager
   - Restore on restart
   - Handle counter corruption (reset to 0)

4. **Security Considerations**
   - Never decrement counter (prevents replay)
   - Validate lookahead bounds (prevent brute force)
   - Clear secrets from memory after use

## 4. CREDENTIAL MANAGER MODULE

### File: `src/credentials/credential-manager.js`

#### Purpose
Secure storage and lifecycle management of MFA secrets and related metadata.

#### Implementation Specification

```javascript
class CredentialManager {
  /**
   * Store MFA credential (secret) securely
   * @param {string} identifier - Domain/service identifier (e.g., 'google.com')
   * @param {string} secret - Base32-encoded secret
   * @param {Object} metadata - { issuer, account, algorithm, digits, period, provider }
   * @returns {Promise<void>}
   */
  async store(identifier, secret, metadata)
  
  /**
   * Retrieve stored credential
   * @param {string} identifier
   * @returns {Promise<Object>} { secret, metadata }
   */
  async retrieve(identifier)
  
  /**
   * List all stored credentials
   * @returns {Promise<Array>} [ { identifier, issuer, account, algorithm, digits, period } ]
   */
  async list()
  
  /**
   * Delete credential
   * @param {string} identifier
   * @returns {Promise<void>}
   */
  async delete(identifier)
  
  /**
   * Check if credential exists
   * @param {string} identifier
   * @returns {Promise<boolean>}
   */
  async exists(identifier)
  
  /**
   * Update credential metadata
   * @param {string} identifier
   * @param {Object} metadata
   * @returns {Promise<void>}
   */
  async updateMetadata(identifier, metadata)
  
  /**
   * Export credentials (encrypted)
   * @returns {Promise<string>} Encrypted JSON export
   */
  async export()
  
  /**
   * Import credentials (encrypted)
   * @param {string} exportedData
   * @param {string} password - Password to decrypt
   * @returns {Promise<void>}
   */
  async import(exportedData, password)
}
```

#### Storage Strategy

1. **Encryption**
   - Encrypt secrets using AES-256-GCM
   - Derive key from master password + salt (PBKDF2)
   - Store only encrypted secrets on disk
   - Keep decrypted secrets in memory during session

2. **Storage Locations**
   ```
   Session profiles:
   ~/.basset-hound/profiles/{profile-name}/credentials.enc
   
   Session state:
   .basset-hound/session-cache/{session-id}/mfa-config.json
   ```

3. **Memory Management**
   - Clear secrets from memory after use
   - Use locked buffers (prevent swapping)
   - Zeroize sensitive data on logout
   - Timeout-based memory clearing

## 5. WEBSOCKET COMMAND INTEGRATION

### File: `websocket/commands/credential-commands.js`

#### WebSocket Command Specifications

##### 1. `generate_totp`
Generate a time-based one-time password for a stored credential.

**Request:**
```json
{
  "command": "generate_totp",
  "params": {
    "identifier": "google.com",
    "preemptive": false
  }
}
```

**Response:**
```json
{
  "success": true,
  "token": "123456",
  "expiresAt": 1686764430000,
  "validFor": 15000,
  "metadata": {
    "issuer": "Google",
    "account": "alice@example.com",
    "algorithm": "SHA1",
    "period": 30
  }
}
```

**Parameters:**
- `identifier` (required): Domain or service identifier
- `preemptive` (optional, default: false): If true, return next token if current expires in <5s

**Response Fields:**
- `token`: 6-8 digit code
- `expiresAt`: UNIX timestamp when token expires
- `validFor`: Milliseconds until expiry
- `metadata`: Issuer and account information

##### 2. `generate_hotp`
Generate a counter-based one-time password.

**Request:**
```json
{
  "command": "generate_hotp",
  "params": {
    "identifier": "backup-codes",
    "increment": true
  }
}
```

**Response:**
```json
{
  "success": true,
  "token": "123456",
  "counter": 5,
  "metadata": {
    "issuer": "Backup",
    "account": "alice@example.com"
  }
}
```

**Parameters:**
- `identifier` (required): Service identifier
- `increment` (optional, default: true): Increment counter after generation

**Response Fields:**
- `token`: Generated code
- `counter`: Current counter value (for resync debugging)

##### 3. `validate_totp`
Validate a TOTP token with drift tolerance.

**Request:**
```json

  "command": "validate_totp",
```

**Likelihood:** Medium | **Impact:** Medium
- **Mitigation:** Support manual URI input as fallback, multiple QR detection methods
- **Testing:** Test against 20+ MFA registration pages

### Risk: Provider Compatibility
**Likelihood:** Medium | **Impact:** Medium
- **Mitigation:** Validate against 5+ providers during development, maintain provider matrix
- **Testing:** Real-world testing with Google, Microsoft, GitHub, Authy, AWS

### Risk: Secret Exposure
**Likelihood:** Low | **Impact:** Critical
- **Mitigation:** Encrypt all secrets at rest, zeroize from memory, no logging
- **Testing:** Security audit, code review for secret handling

### Risk: Counter Out-of-Sync
**Likelihood:** Low | **Impact:** Medium
- **Mitigation:** Implement resync logic with lookahead, detect and recover from drift
- **Testing:** Simulate multiple token generations, verify recovery

## SECTION 1: CRITICAL VULNERABILITIES (CONFIRMED FROM FOLLOW-UP AUDIT)

### CVE-1: Insufficient Entropy in Session ID Generation

**File:** `/src/session/session-manager.js` (Line 27)
**Severity:** 🔴 CRITICAL (CVSS 7.5)
**CWE:** CWE-338 (Weak PRNG)

**Current Code:**
```javascript
const sessionId = crypto.randomBytes(4).toString('hex');
// Only 4 bytes = 32 bits of entropy
```

**Impact:** Session ID brute-force in ~4 billion attempts (2^32)

**Fix:**
```javascript
const sessionId = `session-${crypto.randomBytes(16).toString('hex')}`;
// 16 bytes = 128 bits of entropy
```

**Effort:** 30 minutes
**Risk:** LOW (simple replacement)
**Timeline:** Immediate (Phase 3, Week 1)

## SECTION 2: ADDITIONAL VULNERABILITIES (POST-AUDIT FINDINGS)

### CVE-4: Missing Global Rate Limiting (DoS Risk)

**Affected:** Entire WebSocket server
**Severity:** 🟠 HIGH (CVSS 7.5)
**CWE:** CWE-770 (Allocation Without Limits)

**Current State:**
- Per-client rate limiting: Yes (RateLimitAdapter)
- Global rate limiting: No
- Attack vector: Single attacker spams connections → system overload

**Proposed Fix:**
```javascript
class GlobalRateLimiter {
  constructor(options = {}) {
    this.requestsPerSecond = options.requestsPerSecond || 10000;
    this.windowSize = options.windowSize || 1000;
    this.requestCounts = [];
  }

  isAllowed() {
    const now = Date.now();
    this.requestCounts = this.requestCounts.filter(t => now - t < this.windowSize);

    if (this.requestCounts.length >= this.requestsPerSecond * (this.windowSize / 1000)) {
      return false;
    }

    this.requestCounts.push(now);
    return true;
  }

  getStatus() {
    return {
      currentRate: this.requestCounts.length / (this.windowSize / 1000),
      limit: this.requestsPerSecond,
      status: this.isAllowed() ? 'ok' : 'rate_limited'
    };
  }
}
```

**Effort:** 20-30 hours
**Risk:** LOW (graceful degradation)
**Timeline:** Phase 3, Week 2
**Impact:** Prevents DoS attacks, ensures system stability

### CVE-5: Unencrypted Session Data

**Files:** `/src/session/session-manager.js`, session storage
**Severity:** 🟠 HIGH (CVSS 6.5)
**CWE:** CWE-312 (Cleartext Storage)

**Current State:**
- Session files stored as plaintext JSON
- Sensitive data: cookies, tokens, local storage visible
- Risk: File system compromise exposes user sessions

**Proposed Fix:**
```javascript
const crypto = require('crypto');

class EncryptedSessionManager {
  constructor(encryptionKey) {
    this.cipher = 'aes-256-gcm';
    this.key = crypto.scryptSync(encryptionKey, 'salt', 32);
  }

  encryptSession(sessionData) {
    const iv = crypto.randomBytes(16);
    const cipher = crypto.createCipheriv(this.cipher, this.key, iv);

    let encrypted = cipher.update(JSON.stringify(sessionData), 'utf8', 'hex');
    encrypted += cipher.final('hex');

    const authTag = cipher.getAuthTag();

    return JSON.stringify({
      iv: iv.toString('hex'),
      authTag: authTag.toString('hex'),
      data: encrypted
    });
  }

  decryptSession(encrypted) {
    const { iv, authTag, data } = JSON.parse(encrypted);
    const decipher = crypto.createDecipheriv(
      this.cipher,
      this.key,
      Buffer.from(iv, 'hex')
    );
    decipher.setAuthTag(Buffer.from(authTag, 'hex'));

    let decrypted = decipher.update(data, 'hex', 'utf8');
    decrypted += decipher.final('utf8');

    return JSON.parse(decrypted);
  }
}
```

**Effort:** 40-50 hours
**Risk:** MEDIUM (requires key management)
**Timeline:** Phase 3, Week 3-4
**Impact:** Protects session data at rest

### CVE-6: Missing Audit Logging for Sensitive Operations

**Files:** Entire WebSocket API
**Severity:** 🟠 HIGH (CVSS 6.3)
**CWE:** CWE-778 (Insufficient Logging)

**Current State:**
- No forensic trail of sensitive operations
- No audit log for: credential changes, permission changes, access to sensitive commands
- Risk: Compliance violation, post-incident investigation impossible

**Proposed Fix:**
```javascript
class AuditLogger {
  constructor(options = {}) {
    this.logFile = options.logFile || '/var/log/basset-hound/audit.log';
    this.sensitivePaths = new Set([
      'set_proxy', 'tor_enable', 'execute_script',
      'create_profile', 'delete_profile', 'export_cookies'
    ]);
  }

  logOperation(operation) {
    if (!this.sensitivePaths.has(operation.command)) return;

    const logEntry = {
      timestamp: new Date().toISOString(),
      command: operation.command,
      user: operation.userId || 'anonymous',
      sessionId: operation.sessionId,
      args: this.sanitize(operation.args),
      result: operation.result,
      duration: operation.duration,
      clientIp: operation.clientIp
    };

    // Write to audit log
    fs.appendFileSync(
      this.logFile,
      JSON.stringify(logEntry) + '\n'
    );
  }

  sanitize(args) {
    // Remove sensitive values before logging
    const sanitized = { ...args };
    if (sanitized.password) sanitized.password = '***';
    if (sanitized.token) sanitized.token = '***';
    return sanitized;
  }
}
```

**Effort:** 30-40 hours
**Risk:** LOW (read-only logging)
**Timeline:** Phase 3, Week 2
**Impact:** Forensic trail, compliance readiness

### CVE-7: Selector Injection Risks (XPath/CSS)

**Files:** `/websocket/server.js`, command handlers
**Severity:** 🟡 MEDIUM (CVSS 4.3)
**CWE:** CWE-95 (Code Injection)

**Current State:**
- CSS selectors and XPath validated in forms module
- But not validated in extraction commands
- Risk: Malformed selectors could cause DoS or unexpected behavior

**Proposed Fix:**
```javascript
class SelectorValidator {
  static validateCSSSelector(selector) {
    // Reject if empty or extremely long
    if (!selector || selector.length > 1000) {
      throw new Error('Invalid CSS selector');
    }

    // Try to parse with standard API
    try {
      document.querySelector(selector);
      return true;
    } catch (e) {
      throw new Error(`Invalid CSS selector: ${e.message}`);
    }
  }

  static validateXPath(xpath) {
    if (!xpath || xpath.length > 1000) {
      throw new Error('Invalid XPath');
    }

    // Check for dangerous patterns
    const dangerous = ['contains(@', 'normalize-space(', 'substring'];
    for (const pattern of dangerous) {
      if (xpath.includes(pattern)) {
        // Log warning, but allow (these are valid XPath)
        console.warn(`Potential XPath injection pattern: ${pattern}`);
      }
    }

    // Try to compile XPath
    try {
      document.evaluate(xpath, document, null, XPathResult.FIRST_ORDERED_NODE_TYPE);
      return true;
    } catch (e) {
      throw new Error(`Invalid XPath: ${e.message}`);
    }
  }
}

// Usage in handlers:

// validateXPath(args.xpath);
```

**Effort:** 20-30 hours

### CVE-8: Cache Poisoning (Screenshot Cache)

**Files:** `/screenshots/manager.js`, cache system
**Severity:** 🟡 MEDIUM (CVSS 4.7)
**CWE:** CWE-444 (Inconsistent Interpretation)

**Current State:**
- Screenshot cache by URL
- No freshness validation
- Risk: Old screenshot could be served for updated URL

**Proposed Fix:**
```javascript
class SecureCacheManager {
  constructor() {
    this.cache = new Map();
    this.hashes = new Map(); // Track page hash per URL
  }

  async getScreenshot(url) {
    const cacheKey = url;
    const cached = this.cache.get(cacheKey);

    if (cached) {
      // Validate freshness by comparing page hash
      const currentHash = await this.getPageHash(url);
      if (cached.pageHash === currentHash &&
          Date.now() - cached.timestamp < 5000) {
        return cached.data;
      }
    }

    // Fetch new screenshot
    const screenshot = await this.captureScreenshot(url);
    const pageHash = await this.getPageHash(url);

    this.cache.set(cacheKey, {
      data: screenshot,
      pageHash,
      timestamp: Date.now()
    });

    return screenshot;
  }

  async getPageHash(url) {
    // Compute hash of page content to validate freshness
    const content = await this.getPageContent(url);
    return crypto.createHash('sha256').update(content).digest('hex');
  }
}
```

**Effort:** 20-25 hours
**Risk:** LOW (validation only)
**Timeline:** Phase 3, Week 3
**Impact:** Prevents stale cache serves

### CVE-10: Timing Attacks in Sandbox

**Files:** `/execution/safe-js-executor.js`
**Severity:** 🟡 MEDIUM (CVSS 4.2)
**CWE:** CWE-208 (Observable Timing Discrepancy)

**Current State:**
- Sandbox timeout validation uses string comparison
- Timing variations could leak information about data
- Risk: Attacker could infer data through timing analysis

**Proposed Fix:**
```javascript
const crypto = require('crypto');

class TimingSafeComparison {
  static compare(a, b) {
    // Use constant-time comparison to prevent timing attacks
    return crypto.timingSafeEqual(
      Buffer.from(a || ''),
      Buffer.from(b || '')
    );
  }

  static validateSandboxOutput(output, expected) {
    // Constant-time validation prevents timing leaks
    const outputHash = crypto.createHash('sha256').update(output).digest();
    const expectedHash = crypto.createHash('sha256').update(expected).digest();

    return this.compare(outputHash, expectedHash);
  }
}
```

**Effort:** 10-15 hours
**Risk:** LOW (crypto API usage)
**Timeline:** Phase 3, Week 2
**Impact:** Prevents timing-based information leaks

### CVE-11: Insufficient Sandboxing (Dangerous APIs)

**Files:** `/execution/safe-js-executor.js`
**Severity:** 🟡 MEDIUM (CVSS 5.3)
**CWE:** CWE-95 (Code Injection)

**Current State:**
- Some dangerous APIs still accessible in sandbox
- Risk: Malicious scripts could escape sandbox
- Example: `require()`, `eval()`, global state access

**Proposed Fix:**
```javascript
class StrictSandbox {
  constructor() {
    this.allowlist = new Set([
      'Math', 'Array', 'Object', 'String', 'RegExp', 'JSON',
      'Date', 'Boolean', 'Number', 'isNaN', 'parseFloat', 'parseInt'
    ]);
  }

  execute(code) {
    // Create isolated context with minimal globals
    const context = {};

    // Only expose allowlist globals
    for (const name of this.allowlist) {
      context[name] = global[name];
    }

    // Provide safe console
    context.console = {
      log: (...args) => this.logs.push(args),
      error: (...args) => this.errors.push(args),
      warn: (...args) => this.warns.push(args)
    };

    // Block dangerous globals
    context.require = undefined;
    context.process = undefined;
    context.global = undefined;
    context.__dirname = undefined;
    context.__filename = undefined;

    // Execute in isolated context
    const vm = require('vm');
    return vm.runInNewContext(code, context, {
      timeout: 5000,
      displayErrors: true
    });
  }
}
```

**Effort:** 25-35 hours
**Risk:** MEDIUM (requires comprehensive testing)
**Timeline:** Phase 3, Week 3-4
**Impact:** Prevents sandbox escape

### CVE-12: Error Message Disclosure (Information Leakage)

**Files:** `/websocket/server.js`, all handlers
**Severity:** 🟡 MEDIUM (CVSS 4.3)
**CWE:** CWE-209 (Information Exposure)

**Current State:**
- Full stack traces sent in error responses
- File paths, code details visible to clients
- Risk: Information leakage to attackers

**Proposed Fix:**
```javascript
class ErrorHandler {
  static sanitize(error) {
    // In production, return generic error
    if (process.env.NODE_ENV === 'production') {
      return {
        error: 'Internal server error',
        code: error.code || 'ERR_INTERNAL',
        // No stack trace, no file paths
      };
    }

    // In development, return detailed error
    return {
      error: error.message,
      code: error.code,
      stack: error.stack.split('\n')
    };
  }

  static handleWebSocketError(error, ws) {
    const sanitized = this.sanitize(error);

    // Log full error internally
    logger.error('WebSocket error:', error);

    // Send sanitized error to client
    ws.send(JSON.stringify({
      type: 'error',
      ...sanitized
    }));
  }
}
```

**Effort:** 10-15 hours
**Risk:** LOW (error handling only)
**Timeline:** Phase 3, Week 1
**Impact:** Prevents information leakage

CONFIGURATION & USAGE
================================================================================

Default Configuration:

FailureRecoveryManager:
  - initialBackoff: 100ms
  - maxBackoff: 30000ms (30s)
  - maxRetries: 5
  - enableAutoRecovery: true

SessionHistoryManager:
  - enableSqlite: true (falls back to JSON)
  - retentionDays: 30
  - maxHistorySize: 1000000 bytes per session

CampaignManager:
  - storageDir: /tmp/basset-sessions/campaigns
  - maxParallelSessions: 5
  - retryOnFailure: true
  - errorHandling: 'continue' (alternative: 'abort')

Example Usage:

1. Create Campaign
   WebSocket: create_campaign
   {
     name: "Investigation X",
     maxParallelSessions: 5,
     metadata: { objective: "Target reconnaissance" }
   }

2. Add Sessions to Campaign
   WebSocket: add_campaign_session
   { campaignId: "...", sessionId: "s1", dependencies: [] }

3. Monitor Failures
   WebSocket: detect_failure_type
   { sessionId: "s1", statusCode: 429, headers: {...} }

4. Get Recovery Strategy
   WebSocket: handle_failure
   { sessionId: "s1", failureType: "rate_limit" }

5. Query Session History
   WebSocket: query_session_operations
   { sessionId: "s1", filters: { status: "error", limit: 10 } }

6. Export Results
   WebSocket: export_campaign_results
   { campaignId: "...", format: "json" }

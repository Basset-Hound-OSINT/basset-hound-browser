ARCHITECTURE HIGHLIGHTS
================================================================================

Load Balancing Architecture:
┌─────────────────────────────────────────┐
│  Client Request (port 8765)             │
└────────────┬────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────┐
│  Load Balancer (in-process)             │
│  - Rate limiting (per client IP)        │
│  - Session affinity routing             │
│  - Health-aware selection               │
└────────────┬────────────────────────────┘
             │
      ┌──────┼──────┬──────┐
      ▼      ▼      ▼      ▼
   [BE1]  [BE2]  [BE3]  [BE4]  (Healthy backends)

Session Persistence Architecture:
┌─────────────────────────────────────────┐
│  Application Request                    │
└────────────┬────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────┐
│  Session Store (high-level API)         │
│  - Validation, TTL, consistency         │
└────────────┬────────────────────────────┘
             │
      ┌──────┼──────┐
      ▼      ▼      ▼
   Redis  PostgreSQL  S3 Archive
   (Hot)   (Warm)     (Cold)
   <1ms    10-50ms    1-10s

Data Flow:
[Write] → Redis (immediate) → PostgreSQL (write-through)
[Read]  → Redis (cached) → PostgreSQL (fallback) → Redis (restore)

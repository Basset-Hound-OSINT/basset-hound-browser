## Feature Overview

### What Is Advanced Forensic Analysis?

**Purpose**: Provide intelligent analysis of network traffic, DOM mutations, and media streams.

**Key Innovation**: Goes beyond simple capture (existing NetworkForensicsCollector) to provide analysis and intelligence that external agents can use directly.

### Three Pillars

```
1. NETWORK ANALYSIS (400-600 LOC)
   ↓
   HAR Export (W3C standard)
   Request Classification (tracking detection)
   Network Timeline (dependencies, flow)

2. DOM ANALYSIS (300-400 LOC)
   ↓
   DOM Snapshots (complete tree capture)
   Mutation Detection (change tracking)
   Structure Analysis (composition insights)

3. MEDIA FORENSICS (200-300 LOC)
   ↓
   Metadata Extraction (format, bitrate, duration)
   Stream Analysis (CDN, DRM, latency)
   Content Extraction (URL harvesting)
```

2. **Independence**: Network, DOM, Media are completely separate

## Usage Examples

### Network Analysis Example
```javascript
// Navigate to news site
await navigate('https://news.example.com');
await wait(3000);

// Export as HAR
const har = await send({
  id: 1,
  command: 'get_network_har',
  format: 'json'
});
// Result: 50 requests, all timing data

// Analyze
const analysis = await send({
  id: 2,
  command: 'analyze_network_requests',
  includeTracking: true
});
// Result: 15 tracking domains, 68% third-party
```

### DOM Analysis Example
```javascript
// Snapshot before
const snap1 = await send({
  id: 1,
  command: 'capture_dom_snapshot'
});

// User action
await click('#button');
await wait(2000);

// Snapshot after
const snap2 = await send({
  id: 2,
  command: 'capture_dom_snapshot'
});

// Detect changes
const mutations = await send({
  id: 3,
  command: 'detect_dom_mutations',
  snapshotId1: snap1.data.id,
  snapshotId2: snap2.data.id
});
// Result: 50 additions, 30 removals, 80 total mutations
```

### Media Forensics Example
```javascript
// Extract metadata
const media = await send({
  id: 1,
  command: 'extract_media_metadata',
  type: 'video'
});

// Analyze streams
for (const video of media.data) {
  const analysis = await send({
    id: 2,
    command: 'analyze_streams',
    url: video.url
  });
  // Result: 1920x1080 H.264, CloudFlare CDN, Widevine DRM
}
```

## Key Decision Points

### Why Three Separate Modules?
**Answer**: Independence enables:
- Parallel development of all three
- Testing each independently
- Using each without others
- Easy to add more modules later

### Why Build on NetworkForensicsCollector?
**Answer**: Avoids duplication:
- Network capture already exists (1,267 LOC)
- HAR Exporter converts format only
- Request Analyzer adds intelligence
- Cleaner separation of concerns

### Why WebSocket Commands vs. Direct API?
**Answer**: Integration pattern:
- Consistent with 164 existing commands
- Remote agents can use directly
- Command dispatcher handles routing
- Response serialization reused

### Why Not Include Intelligence in Capture?
**Answer**: Architectural principle:
- Browser is capture/control tool
- Intelligence happens in agents
- Cleaner abstraction
- Reusable across agents

## What's NOT Included

### Out of Scope
- Machine learning analysis (agent responsibility)
- Automatic threat detection (agent responsibility)
- Report generation (template responsibility)
- Long-term storage (external DB responsibility)
- Real-time alerting (orchestration responsibility)

### Why Not?
These are higher-level concerns that belong in external agents, not the browser. The browser provides **data and analysis**, agents provide **intelligence**.

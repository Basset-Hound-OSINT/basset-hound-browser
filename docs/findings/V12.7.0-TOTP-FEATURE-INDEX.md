## KEY CONCEPTS EXPLAINED

### TOTP (Time-based One-Time Password)
- **Standard:** RFC 6238
- **Algorithm:** HMAC-SHA1 (default), SHA-256, SHA-512
- **Key:** Base32-encoded secret
- **Counter:** Current Unix time / window (typically 30 seconds)
- **Output:** 6-8 digit code
- **Drift Tolerance:** ±1-2 time windows (±30-60 seconds)
- **Providers:** Google Authenticator, Microsoft, GitHub, Authy, AWS

### HOTP (HMAC-based One-Time Password)
- **Standard:** RFC 4226
- **Algorithm:** HMAC-SHA1
- **Key:** Base32-encoded secret
- **Counter:** Client-incremented counter (0, 1, 2, ...)
- **Output:** 6-8 digit code
- **Resync:** Lookahead to find matching token
- **Use:** Backup codes, hardware tokens

### QR Code Parsing
- **Format:** Standard QR code containing otpauth:// URI
- **Example URI:** `otpauth://totp/Example:alice@example.com?secret=...&issuer=...`
- **Extraction:** Use jsQR library for decoding
- **Fallback:** Manual URI input

### Credential Storage
- **Encryption:** AES-256-GCM at rest
- **Key Derivation:** PBKDF2 (master password + salt)
- **Location:** Session profile or encrypted cache
- **Lifetime:** Persist across browser restarts

## Usage Example (Network Analysis)

```javascript
// Navigate and capture
await navigate('https://news.example.com');
await wait(3000); // Page load

// Export as HAR
const har = await send({
  id: 1,
  command: 'get_network_har',
  format: 'json',
  pretty: true
});
// → 50 requests, all timing data, all headers

// Analyze requests
const analysis = await send({
  id: 2,
  command: 'analyze_network_requests',
  includeTracking: true
});
// → 15 tracking domains, 68% third-party
```

## Usage Example (DOM Analysis)

```javascript
// Capture initial state
const snap1 = await send({
  id: 1,
  command: 'capture_dom_snapshot',
  includeStyles: true
});

// User action
await click('#button');
await wait(2000);

// Capture new state
const snap2 = await send({
  id: 2,
  command: 'capture_dom_snapshot',
  includeStyles: true
});

// Detect changes
const mutations = await send({
  id: 3,
  command: 'detect_dom_mutations',
  snapshotId1: snap1.data.id,
  snapshotId2: snap2.data.id
});
// → 50 additions, 30 removals, 80 total changes
```

## Usage Example (Media Forensics)

```javascript
// Extract all media
const media = await send({
  id: 1,
  command: 'extract_media_metadata',
  type: 'video'
});

// Analyze streams
for (const video of media.data) {
  const analysis = await send({
    id: 2,
    command: 'analyze_streams',
    url: video.url,
    detailed: true
  });
  // → Resolution, codec, CDN, DRM, latency
}
```

## Common Pitfalls to Avoid

❌ **Don't**: Make modules dependent on each other  
✅ **Do**: Keep Network, DOM, Media completely independent

❌ **Don't**: Store entire snapshots without pruning  
✅ **Do**: Implement snapshot limits and GC

❌ **Don't**: Assume network data always complete  
✅ **Do**: Handle missing timing/header data gracefully

❌ **Don't**: Capture full DOM text for large pages  
✅ **Do**: Truncate text, capture first 100 chars

❌ **Don't**: Hardcode tracking domain list  
✅ **Do**: Use configurable, updateable database

## Performance Optimization Tips

**HAR Export**: Stream large exports, use compression  
**DOM Snapshots**: Use element IDs, not full objects  
**Mutations**: Cache previous state, optimize diff  
**Metadata**: Parallelize extraction, use requestIdleCallback  
**Memory**: Aggressive GC between snapshots, implement pruning

REQUIREMENT 4: HMAC MESSAGE AUTHENTICATION
Status: ❌ NOT IMPLEMENTED
Expected: All messages signed with HMAC-SHA256 via HMACSignerMessage
Found: Zero message authentication in Wave 14

Affected Operations:
- add_monitor request/response
  Current: Plain JSON over WebSocket
  Required: HMAC envelope with signature, timestamp, nonce

- get_changes request/response
  Current: Plain JSON
  Required: Signed envelope

- configure_proxy request/response
  Current: Plain JSON
  Required: Signed envelope, credential redaction

- restore_session request/response
  Current: Plain JSON
  Required: Signed envelope, ownership verification

Implementation Pattern:

  // Client side:
  const envelope = signer.createAuthenticatedMessage({
    command: 'add_monitor',
    url: 'https://example.com',
    changeTreshold: 5
  });
  ws.send(JSON.stringify(envelope));

  // Server side:
  const verification = signer.verifyMessage(envelope);
  if (!verification.valid) {
    return { error: 'Message verification failed' };
  }
  const data = verification.data;
  processCommand(data);

Envelope Format:
  {
    payload: { command, params },
    signature: "hex-string",
    timestamp: 1717000000000,
    nonce: "random-hex-string"
  }

Benefits:
- Prevents tampering with commands
- Detects replay attacks (via timestamp)
- Ensures message integrity
- Detects man-in-the-middle

Effort to Implement: 4-5 hours
Blocker: YES (for v12.2.0 enterprise release)

REQUIREMENT 5: PATH TRAVERSAL PREVENTION
Status: ⚠️  PARTIALLY IMPLEMENTED
Expected: All file operations use PathValidator.validatePath()
Found: Session persistence creates directories without safe directory enforcement

Affected Code:

  // Current (unsafe):
  ensureStorageDir() {
    if (!fs.existsSync(this.storageDir)) {
      fs.mkdirSync(this.storageDir, { recursive: true });  // ❌ User-supplied path
    }
  }

  // Should be (safe):
  ensureStorageDir() {
    const result = PathValidator.ensureSafeDir('sessions', appBaseDir);
    if (!result.success) throw new Error(result.error);
    this.storageDir = result.path;  // ~.basset-hound/sessions/
  }

Safe Directory Structure:
  ~.basset-hound/
    ├── sessions/          (session files)
    ├── snapshots/         (monitoring snapshots)
    ├── screenshots/       (captured images)
    ├── recordings/        (session recordings)
    └── exports/           (data exports)

All session files must be stored under:
  $HOME/.basset-hound/sessions/

Validation Rules:
- No path traversal (../ not allowed)
- No symlinks
- No absolute paths
- Reject paths with null bytes
- Filenames sanitized of special chars

Affected Methods:
  - SessionPersistence.ensureStorageDir()
  - SessionPersistence.saveSession()
  - SessionPersistence.saveSnapshot()
  - MonitoringService (if storing snapshots)

Effort to Implement: 2-3 hours
Blocker: YES - Critical for file security

REQUIREMENT 6: SENSITIVE DATA CLEANING AND MASKING
Status: ❌ INSUFFICIENT IMPLEMENTATION
Expected: All sensitive data masked via DataCleaner before output/logging
Found: Credentials logged in plaintext, no data sanitization

Sensitive Data in Wave 14:

1. Proxy Credentials
   Current: "address": "user:password@proxy.com:8080"  ❌
   Required: "address": "proxy.com:8080" (mask creds)  ✅

2. Session Cookies
   Current: "cookies": { "session_token": "abc123def456..." }  ❌
   Required: "cookies": { "session_token": "abc...456" }  ✅

3. Geographic Location
   Current: Exposed in session history  ⚠️
   Required: Mask specific locations, aggregate only  ⚠️

4. Alert Payloads
   Current: Full change data including URLs, details  ⚠️
   Required: Summarized changes without full data  ✅

5. Version Strings
   Current: Full version in detection output  ⚠️
   Required: Version string validation + escaping  ✅

Implementation:

  // Before returning monitor config:
  const response = dataCleaner.sanitizeObject(monitor);
  // Masks: webhook URLs (partial), API keys, credentials

  // Before logging errors:
  const safeError = dataCleaner.sanitizeError(error);
  // Masks: paths, credentials, internal IPs

  // Before sending alert:
  const safeMessage = dataCleaner.sanitizeObject(alertData);
  // Masks: API keys, credentials, sensitive details

Masking Rules:
  Password/Secret         → "***"
  Token/API Key           → "sk_l...abc123"
  Cookie Value            → "abc...123"
  URL with Credentials    → Strip creds
  Email Address           → "u***@example.com"
  Phone Number            → "****-123-4567"
  Session ID              → "sess...4ab"
  Proxy Address           → "1.2.3.4:8080" (no creds)

Effort to Implement: 4-5 hours
Blocker: YES - Data leakage risk

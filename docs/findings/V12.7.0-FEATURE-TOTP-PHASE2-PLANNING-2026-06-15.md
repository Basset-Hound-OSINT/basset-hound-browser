### Part 1: WebSocket Command Handlers (Day 1)

#### Task 3-1a: Implement WebSocket Commands

**File:** `websocket/server.js` (update)

**5 New Commands:**

1. **`generate_totp`** - Generate time-based OTP
   ```javascript
   {
     "command": "generate_totp",
     "params": {
       "secret": "base32-encoded-secret",
       "algorithm": "sha1|sha256|sha512" (optional, default: "sha1"),
       "digits": 6|7|8 (optional, default: 6),
       "window_size": 30|60 (optional, default: 30),
       "time": 1234567890 (optional, current time if omitted)
     },
     "response": {
       "token": "123456",
       "remaining_seconds": 28,
       "valid_from": 1234567870,
       "valid_until": 1234567900
     }
   }
   ```

2. **`generate_hotp`** - Generate counter-based OTP
   ```javascript
   {
     "command": "generate_hotp",
     "params": {
       "secret": "base32-encoded-secret",
       "counter": 42,
       "algorithm": "sha1|sha256|sha512" (optional, default: "sha1"),
       "digits": 6|7|8 (optional, default: 6),
       "lookahead": 1 (optional, for server mismatch)
     },
     "response": {
       "token": "987654",
       "counter": 42,
       "lookahead_tokens": ["987653", "987652"]
     }
   }
   ```

3. **`parse_mfa_qr`** - Extract secret from QR code
   ```javascript
   {
     "command": "parse_mfa_qr",
     "params": {
       "image_base64": "iVBORw0KGgo...",
       "or_image_path": "/path/to/qr.png"
     },
     "response": {
       "secret": "JBSWY3DPEBLW64TMMQ",
       "issuer": "Google",
       "account": "user@example.com",
       "algorithm": "SHA1",
       "digits": 6,
       "period": 30
     }
   }
   ```

4. **`fill_mfa_code`** - Auto-fill OTP into form
   ```javascript
   {
     "command": "fill_mfa_code",
     "params": {
       "token": "123456",
       "selector": "input[id='mfa-code']" (optional, auto-detect if omitted),
       "submit": true|false (optional, default: false),
       "submit_selector": "button.submit" (optional)
     },
     "response": {
       "filled": true,
       "selector_used": "input[id='mfa-code']",
       "submitted": false
     }
   }
   ```

5. **`get_mfa_status`** - Check MFA setup on current site
   ```javascript
   {
     "command": "get_mfa_status",
     "response": {
       "mfa_detected": true,
       "mfa_type": "google_authenticator|microsoft|github|authy|generic",
       "form_selector": "form.login-2fa",
       "input_selector": "input[id='totp']",
       "issuer": "GitHub",
       "account": "octocat"
     }
   }
   ```

### Part 2: 2FA Automation Module (Day 1-2)

#### Task 3-2a: MFA Automator Engine

**File:** `src/credentials/mfa-automator.js` (new)

**Purpose:** Orchestrate full 2FA workflow - detection, generation, filling, submission

**Workflow:**
1. Detect MFA screen on current page
2. Identify provider type and OTP field
3. Generate correct OTP code
4. Fill OTP into input field
5. Submit form (if configured)
6. Validate success (check for error messages)
7. Handle retry on failure

**Implementation:**
```javascript
class MFAAutomator {
  async autoFill2FA(options = {})
    // Main entry point
    // options: {submit: true, max_retries: 3, timeout: 10000}
    // Returns: {success: true/false, provider, token, error}
  
  async detectMFAScreen()
    // Returns: {detected: true/false, provider, details}
  
  async generateOTP(provider, ...config)
    // Returns: {token, valid_until}
  
  async fillOTPField(token, selector)
    // Returns: {filled: true/false, error}
  
  async validateSubmission()
    // Check for error messages, success indicators
    // Returns: {success: true/false, details}
  
  async retryWithClockSkew()
    // Handle time sync issues (±30 seconds)
    // Returns: {success: true/false}
}
```

**Error Handling:**
- Clock skew: Try ±30 seconds automatically
- Failed form fill: Retry with different selectors
- Provider detection failure: Ask user or fallback
- Network timeout: Retry up to 3 times
- Invalid token: Retry with time correction

#### Task 4-7: Troubleshooting Guide

**File:** `docs/guides/mfa-troubleshooting.md` (new)

**Common Issues:**

1. **"Invalid token" Error**
   - Cause: Clock skew or time sync issue
   - Solution: Check system time, use ±30 second window
   - Command: Automatic retry with clock correction

2. **"Form field not found" Error**
   - Cause: Non-standard form structure
   - Solution: Use custom selector or manual fill
   - Workaround: Use get_mfa_status first

3. **Provider Not Detected**
   - Cause: Unknown provider or page structure
   - Solution: Specify provider manually
   - Fallback: Generic TOTP generator

4. **QR Code Parsing Fails**
   - Cause: Low image quality or corrupted QR
   - Solution: Use higher resolution screenshot
   - Alternative: Manual secret entry

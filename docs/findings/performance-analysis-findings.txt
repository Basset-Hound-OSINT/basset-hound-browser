BOTTLENECK ANALYSIS - 5 CRITICAL ISSUES IDENTIFIED
─────────────────────────────────────────────────────────────────────────────

BOTTLENECK #1: SCREENSHOT ENCODING (CRITICAL) ⚠️
  Severity:  CRITICAL
  Impact:    12.1% of CPU, 15-20% of total latency
  Root Cause: Synchronous WebP encoding in main event loop
  Status:    OPT-08 planned but NOT implemented
  Solution:  Parallel GPU buffers + worker thread pool
  Expected:  -67% latency reduction (150ms → 50-60ms)
  Effort:    6-8 hours

BOTTLENECK #2: PRIORITY QUEUE NOT INTEGRATED (HIGH) ⚠️
  Severity:  HIGH
  Impact:    P99 latency +40-70% at high concurrency
  Root Cause: Priority queue implemented (333 lines) but not activated
  Status:    OPT-09 component exists, not wired to handlers
  Solution:  Activate PriorityQueue in ConnectionPool
  Expected:  -41% P99 latency reduction (1.7ms → 1.0ms)
  Effort:    3-4 hours (integration only)

BOTTLENECK #3: DOM TRAVERSAL WITHOUT CACHE (MEDIUM-HIGH) ⚠️
  Severity:  MEDIUM-HIGH
  Impact:    10-30ms per extraction, 5-10% of extraction time
  Root Cause: DOMExtractionCache implemented (176 lines) but not called
  Status:    OPT-13 cache exists, handlers not using it
  Solution:  Wire cache into text/HTML/form/link extraction handlers
  Expected:  -50% latency for repeated queries (20-30ms → 1-2ms)
  Effort:    4-5 hours (integration only)

BOTTLENECK #4: RECORDING MEMORY ACCUMULATION (MEDIUM) ⚠️
  Severity:  MEDIUM
  Impact:    80-100MB per long session, memory ceiling at 8+ hours
  Root Cause: StreamingSessionRecorder implemented but handlers use old recorder
  Status:    OPT-11 streaming recorder exists, not activated in session manager
  Solution:  Replace old recorder with streaming recorder (disk writes)
  Expected:  -80% memory reduction for long sessions (50-100MB → 10-15MB)
  Effort:    2-3 hours (activation only)

BOTTLENECK #5: FINGERPRINT TEMPLATE CACHING INACTIVE (MEDIUM) ⚠️
  Severity:  MEDIUM
  Impact:    Session startup 100-150ms (OPT-12 could save 50-70ms)
  Root Cause: FingerprintTemplateCache implemented (203 lines) but not integrated
  Status:    OPT-12 cache exists, evasion system generates fresh each time
  Blocker:   Requires evasion effectiveness validation (security review)
  Solution:  Integrate cache with session init, validate effectiveness
  Expected:  -40% fingerprinting latency (100-120ms → 50-70ms)
  Effort:    4-6 hours (implementation + testing)

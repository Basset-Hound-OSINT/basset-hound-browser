## WHY THE PIVOT MAKES SENSE

### User Hierarchy of Needs

```
1. MISSION SUCCESS (Don't get detected)     ← ANONYMITY CRITICAL
   │ Fingerprinting detection = investigation failure
   │ User agent detection = account lockout
   │ IP detection = exposure
   │
2. OPERATIONAL RELIABILITY (App doesn't crash)
   │ Graceful shutdown = nice to have
   │ Health checks = nice to have
   │
3. DEVELOPMENT CONVENIENCE (Dependencies secure)
   │ npm vulnerabilities = cosmetic issue
   │ Deprecated packages = warning to ignore
   └─ Input validation = defensive programming (not user-facing)
```

**Key Insight:** Vulnerabilities in development dependencies don't affect production browser behavior. Fingerprinting detection does.

   - 20+ error scenario tests

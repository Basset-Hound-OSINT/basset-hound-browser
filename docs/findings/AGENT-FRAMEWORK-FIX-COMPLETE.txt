  2. /tests/security/phase2-forensic-entropy.test.js (25 Orchestrator-related tests)

Root Cause Identified:
The Orchestrator class export in /src/agents/orchestrator.js was using a named export
format that was inconsistent with how it was being imported in test files.

File: /src/agents/orchestrator.js
Line 479: module.exports = { Orchestrator };  [INCORRECT]
Expected: module.exports = Orchestrator;      [CORRECT]

The test files were importing with:
  - /tests/agents/orchestration.test.js: const Orchestrator = require('../../src/agents/orchestrator');
  - /tests/security/phase2-forensic-entropy.test.js: const { Orchestrator } = require('../../src/agents/orchestrator');

This created a mismatch where the first file expected the default export of the class,
but received an object with an Orchestrator property instead.

FIXES APPLIED
=============

Fix 1: /src/agents/orchestrator.js (Line 479)
  Changed: module.exports = { Orchestrator };
  To:      module.exports = Orchestrator;

  This makes the export consistent with the default class export pattern used by
  the other agent integration files (ForensicIntegration, OSINTIntegration).

Fix 2: /tests/security/phase2-forensic-entropy.test.js (Line 9)
  Changed: const { Orchestrator } = require('../../src/agents/orchestrator');
  To:      const Orchestrator = require('../../src/agents/orchestrator');

  This updates the import to use the default export syntax, matching the
  orchestration.test.js import pattern.

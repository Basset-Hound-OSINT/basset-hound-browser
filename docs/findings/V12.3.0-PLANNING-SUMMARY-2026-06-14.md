## TEAM STRUCTURE

**Sequential Execution Model:** One phase at a time, clear handoffs

- **Dev Agent 1:** Phases 1-2 (Stability + Features)
- **Dev Agent 2:** Phases 3-4 (Performance + DevOps)
- **QA Agent:** Phase 5 (Regression testing + release approval)

**Why Sequential?**
- Prevents test cycle chaos (1 test run per phase instead of 50+)
- Clear dependencies between phases
- Predictable milestones
- Easier debugging

---

## TEST STRATEGY (Golden Rule)

**Run tests ONCE per phase completion, not after every agent action.**

- Phase 1: 50+ tests on Aug 8 (100% pass required)
- Phase 2: 80+ tests on Aug 15 (85%+ pass required)
- Phase 3: 50+ tests on Aug 15 (performance validation)
- Phase 4: 40+ tests on Aug 21 (infrastructure validation)
- Phase 5: 11,000+ tests on Aug 22 (full regression suite)

**Why?** Prevents endless test cycles that duplicate effort and cause false negatives.

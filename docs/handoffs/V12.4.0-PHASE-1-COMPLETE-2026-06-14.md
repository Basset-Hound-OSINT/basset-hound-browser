## Module 1: Hardware Fingerprint Spoofing

**File:** `src/anonymity/hardware-fingerprint-spoofing.js` (436 LOC)

### Purpose
Overrides navigator properties to spoof hardware identity with session-wide consistency and realistic value ranges.

### Features

#### Properties Spoofed (13 total)
- `navigator.hardwareConcurrency` - CPU cores (2-32)
- `navigator.deviceMemory` - RAM in GB (2-64)
- `navigator.maxTouchPoints` - Touch support (0-10)
- `navigator.vendor` - Browser vendor string
- `navigator.languages` - Language array
- `navigator.language` - Primary language
- `screen.width` / `screen.height` - Display resolution
- `screen.availWidth` / `screen.availHeight` - Usable display space
- `screen.colorDepth` / `screen.pixelDepth` - Color depth (24 or 32)
- `window.devicePixelRatio` - DPI ratio (0.75, 1.0, 1.5, 2.0, 2.5, 2.75, 3.0)
- `Intl.DateTimeFormat().resolvedOptions().timeZone` - Timezone

#### Key Characteristics
- **Session Consistency:** All values remain constant within a session
- **Realistic Ranges:** Values conform to actual device hardware specifications
- **Profile-Based:** Accepts device profiles for consistent identity
- **Random Generation:** Can auto-generate realistic random values
- **Type Safety:** Comprehensive validation of all property values
- **Injection Script:** Generates JavaScript for browser context execution

## Module 3: WebSocket Commands Integration

**File:** `websocket/commands/anonymity-commands.js` (525 LOC)

### Purpose
Provides WebSocket API commands for browser-based hardware spoofing and profile management.

### Commands (9 total)

#### 1. `set_anonymity_profile`
Set hardware spoofing using a pre-built profile.
```javascript
// Request
{ profileName: "iPhone 15 Pro", tabId?: "tab1" }

// Response
{ success: true, profile: { name, deviceType, hardwareConcurrency, ... } }
```

#### 2. `set_anonymity_custom`
Set custom hardware spoofing values.
```javascript
// Request
{ 
  properties: {
    hardwareConcurrency: 8,
    deviceMemory: 16,
    screenWidth: 1920,
    ...
  },
  tabId?: "tab1"
}

// Response
{ success: true, values: { ... } }
```

#### 3. `get_anonymity_profile`
Get current anonymity profile.
```javascript
// Request
{ tabId?: "tab1" }

// Response
{ success: true, profile: { ... } }
```

#### 4. `list_anonymity_profiles`
List available device profiles.
```javascript
// Request
{ deviceType?: "mobile" }

// Response
{ success: true, count: 12, profiles: [...] }
```

#### 5. `create_custom_profile`
Create a new custom device profile.
```javascript
// Request
{ name: "MyDevice", config: { ... } }

// Response
{ success: true, profile: {...}, message: "..." }
```

#### 6. `delete_custom_profile`
Delete a custom profile.
```javascript

// Response

```

#### 7. `get_anonymity_statistics`
Get profile inventory statistics.
```javascript

```

#### 8. `export_profile`
Export a profile as JSON.
```javascript
// Request

// Response
{ success: true, json: "..." }
```

#### 9. `import_profile`
Import a profile from JSON.
```javascript

// Response

```

#### 10. `get_random_profile` (Bonus)
Get random profile with optional device type filter.
```javascript

// Response
```

#### 11. `reset_anonymity_settings` (Bonus)
Reset anonymity settings for a tab.
```javascript

// Response
{ success: true, message: "..." }
```

## Integration Points

### How to Use Hardware Fingerprinting

#### 1. In JavaScript (Browser Context)
```javascript
// Set a device profile
ws.send(JSON.stringify({
  command: 'set_anonymity_profile',
  params: { profileName: 'iPhone 15 Pro' }
}));
```

#### 2. With Custom Values
```javascript
ws.send(JSON.stringify({
  command: 'set_anonymity_custom',
  params: {
    properties: {
      hardwareConcurrency: 8,
      deviceMemory: 16,
      screenWidth: 1920,
      screenHeight: 1080
    }
  }
}));
```

#### 3. Multi-Tab Support
```javascript
// Tab 1: iPhone profile
ws.send(JSON.stringify({
  command: 'set_anonymity_profile',
  params: { profileName: 'iPhone 15', tabId: 'tab1' }
}));

// Tab 2: Windows profile
ws.send(JSON.stringify({
  command: 'set_anonymity_profile',
  params: { profileName: 'Windows Laptop', tabId: 'tab2' }
}));
```

### Integration with WebSocket Server

The commands are registered in `websocket/server.js`:
```javascript
const {
  registerAnonymityCommands
} = require('./commands/anonymity-commands');

// In server initialization
registerAnonymityCommands(commandHandlers, executeInRenderer);
```

## Known Limitations & Future Enhancements

### Current Limitations
1. **No Session Recording:** Hardware values don't persist across browser restarts
2. **Single-Instance Generator:** Device profiles stored in memory only
3. **No Profile Persistence:** Custom profiles lost on server restart
4. **Basic Validation:** No cross-property consistency checking (e.g., RAM vs CPU cores)

### Planned Enhancements
1. **Profile Persistence:** Database storage for custom profiles
2. **Consistency Checking:** Validate cross-property realistic combinations
3. **Profile Randomization:** Periodic drift in fingerprint values
4. **Usage Analytics:** Track which profiles are most effective
5. **A/B Testing:** Support for profile variants for experimentation

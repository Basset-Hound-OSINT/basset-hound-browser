## Quick Start

### Option 1: Automated Deployment (Recommended)

```bash
cd /home/devel/basset-hound-browser

# Run deployment script (builds Docker image, starts container)
./scripts/deploy.sh

# Verify deployment
curl -s http://localhost:8765 | head -c 100
```

**Expected Output:**
```
WebSocket server, expecting client connection
```

### Option 2: Manual Docker Deployment

```bash
# Build Docker image
docker build -t basset-hound-browser:v12.1.0 \
  -f config/docker/Dockerfile .

# Create Docker network (if needed)
docker network create basset-hound-browser

# Run container
docker run -d --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e DISPLAY=:99 \
  -e ELECTRON_DISABLE_SANDBOX=1 \
  --restart unless-stopped \
  basset-hound-browser:v12.1.0

# Check logs
docker logs -f basset-hound-browser
```

## Pre-Deployment Checklist

### Infrastructure Verification

```bash
# 1. Check system resources
free -h                                    # RAM available
df -h /                                    # Disk space
nproc                                      # CPU cores

# 2. Verify Docker installation
docker --version                           # Docker version
docker ps                                  # Docker daemon running

# 3. Check port availability
netstat -tlnp | grep 8765                 # Port 8765 free

# 4. Verify network connectivity
ping 8.8.8.8                               # Internet connectivity
curl https://api.github.com                # DNS resolution
```

### Application Verification

```bash
# 1. Check npm dependencies
npm list                                   # Dependencies installed
npm audit                                  # Security vulnerabilities

# 2. Verify code integrity
node -v                                    # Node version
npm -v                                     # npm version
git status                                 # Code committed

# 3. Check configuration
ls -la config/                             # Config files exist
cat config/docker/Dockerfile              # Dockerfile readable

# 4. Verify test suite
npm test -- --listTests | wc -l           # Tests discoverable
```

```bash
# 1. Check for exposed secrets
```

## Docker Build & Deployment

### Step 1: Build Docker Image

```bash
cd /home/devel/basset-hound-browser

# Build image (without cache for fresh build)
docker build -t basset-hound-browser:v12.1.0 \
  -f config/docker/Dockerfile \
  --no-cache \
  .

# Alternative: Build with cache (faster, for rebuilds)
docker build -t basset-hound-browser:v12.1.0 \
  -f config/docker/Dockerfile \
  .
```

**Build Output:**
- Expected time: 5-10 minutes
- Expected size: 2.5-3.0 GB
- Expected steps: 40-50 build steps

### Step 2: Verify Image

```bash
# List built images
docker images | grep basset-hound-browser

# Inspect image
docker inspect basset-hound-browser:v12.1.0 | jq '.[] | {RepoTags, Size, Created}'

# Expected output:
# REPOSITORY                   TAG         IMAGE ID       SIZE        CREATED
# basset-hound-browser         v12.1.0     abc123def456   2.64GB      2 hours ago
```

### Step 3: Create Docker Network

```bash
# Create isolated network for container communication
docker network create basset-hound-browser

# Verify network
docker network ls | grep basset-hound-browser
```

### Step 4: Start Container

```bash
# Start container
docker run -d \
  --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e DISPLAY=:99 \
  -e ELECTRON_DISABLE_SANDBOX=1 \
  -e LOG_LEVEL=info \
  --restart unless-stopped \
  --memory 4g \
  --cpus 2 \
  basset-hound-browser:v12.1.0

# Verify container started
docker ps | grep basset-hound-browser
```

**Container Configuration Explanation:**
- `-d` - Run in background (detached)
- `--name` - Container name for easy reference
- `--network` - Connect to custom network
- `-p 8765:8765` - Map port 8765 (host:container)
- `-e DISPLAY=:99` - Virtual display for Electron
- `-e ELECTRON_DISABLE_SANDBOX=1` - Disable sandbox mode (required for Electron)
- `-e LOG_LEVEL=info` - Set logging level
- `--restart unless-stopped` - Auto-restart on failure
- `--memory 4g` - Memory limit
- `--cpus 2` - CPU limit

### Step 5: Monitor Startup

```bash
# Watch container logs (Ctrl+C to stop)
docker logs -f basset-hound-browser

# Expected startup sequence (30-60 seconds):
# 1. [INFO] Starting Basset Hound Browser v12.1.0
# 2. [INFO] Initializing WebSocket server...
# 3. [INFO] Loading configuration...
# 4. [INFO] Initializing Electron...
# 5. [INFO] Starting virtual display (Xvfb)...
# 6. [INFO] WebSocket server listening on ws://0.0.0.0:8765
# 7. [INFO] Ready to accept connections
```

## Verification & Smoke Tests

### Test 1: Container Health Check

```bash
# Check container status
docker ps --filter name=basset-hound-browser

# Expected output: STATUS should show "Up" (not "Exited")
```

### Test 2: WebSocket Connectivity

```bash
# Check HTTP upgrade response (WebSocket requirement)
curl -s -i -N \
  -H "Connection: Upgrade" \
  -H "Upgrade: websocket" \
  http://localhost:8765 | head -10

# Expected: HTTP 426 (Upgrade Required) or WebSocket handshake
```

### Test 3: Basic WebSocket Command

```bash
# Create test client script
cat > /tmp/test_websocket.js << 'EOF'
const WebSocket = require('ws');
const ws = new WebSocket('ws://localhost:8765');

ws.on('open', () => {
  console.log('✓ Connected to WebSocket');
  
  // Send ping command
  ws.send(JSON.stringify({
    id: 1,
    command: 'ping'
  }));
});

ws.on('message', (data) => {
  const response = JSON.parse(data);
  console.log('✓ Received response:', response);
  console.log(response.result === 'pong' ? '✓ Ping successful!' : '✗ Unexpected response');
  process.exit(0);
});

ws.on('error', (err) => {
  console.error('✗ Connection error:', err);
  process.exit(1);
});

setTimeout(() => {
  console.error('✗ Timeout - no response');
  process.exit(1);
}, 5000);
EOF

# Run test
cd /home/devel/basset-hound-browser
node /tmp/test_websocket.js
```

### Test 5: Performance Baseline

```bash
# Create load test script
cat > /tmp/perf_test.js << 'EOF'
const WebSocket = require('ws');

async function runTest() {
  const iterations = 100;
  const startTime = Date.now();
  
  for (let i = 0; i < iterations; i++) {
    const ws = new WebSocket('ws://localhost:8765');
    
    await new Promise((resolve, reject) => {
      ws.on('open', () => {
        ws.send(JSON.stringify({ id: i, command: 'ping' }));
      });
      ws.on('message', () => {
        ws.close();
        resolve();
      });
      ws.on('error', reject);
      setTimeout(() => reject(new Error('Timeout')), 5000);
    });
  }
  
  const duration = Date.now() - startTime;
  const throughput = (iterations / duration * 1000).toFixed(2);
  console.log(`✓ ${iterations} requests in ${duration}ms`);
  console.log(`✓ Throughput: ${throughput} req/sec`);
}

runTest().catch(err => {
  console.error('✗ Test failed:', err);
  process.exit(1);
});
EOF

# Run performance test
node /tmp/perf_test.js
```

## Troubleshooting

### Container Fails to Start

```bash
# Check error logs
docker logs basset-hound-browser

# Common issues:

# 1. Port already in use
sudo netstat -tlnp | grep 8765
# Solution: Kill process or use different port

# 2. Insufficient memory
free -h
# Solution: Increase system memory or reduce other containers

# 3. Docker daemon not running
systemctl status docker
# Solution: Start Docker service
sudo systemctl start docker

# 4. Image build failed
docker build --no-cache -f config/docker/Dockerfile .
# Check for network issues or dependency problems
```

### WebSocket Connection Fails

```bash
# Test network connectivity
curl -v http://localhost:8765

# Check if port is listening
ss -tlnp | grep 8765

# Check container network
docker network inspect basset-hound-browser

# Test from inside container
docker exec basset-hound-browser curl http://localhost:8765

# Common solutions:
# 1. Restart container: docker restart basset-hound-browser
# 2. Check firewall: sudo ufw allow 8765
# 3. Verify network: docker network ls
```

### High Memory Usage

```bash
# Monitor memory in detail
docker stats --no-stream basset-hound-browser

# Check for memory leaks
docker top basset-hound-browser

# Restart container to reset memory
docker restart basset-hound-browser

# Check for process leaks
docker exec basset-hound-browser ps aux | wc -l
```

### Slow Performance

```bash
# Check CPU usage
docker stats basset-hound-browser --no-stream | grep CPU

# Check disk I/O
docker stats basset-hound-browser --format "{{.Container}} {{.BlockIO}}"

# Check network connectivity
docker exec basset-hound-browser ping -c 4 8.8.8.8

# Solutions:
# 1. Allocate more CPU/Memory
# 2. Check host system resources (df -h, free -h)
# 3. Restart container
# 4. Check for network bottlenecks
```

### Logs Not Appearing

```bash
# Check if container is writing logs
docker logs basset-hound-browser

# Check log driver
docker inspect basset-hound-browser | grep LogDriver

# Force log output
docker exec basset-hound-browser tail -f /var/log/basset-hound.log

# Increase logging level
docker stop basset-hound-browser
docker rm basset-hound-browser
docker run -d ... -e LOG_LEVEL=debug ... basset-hound-browser:v12.1.0
```

## Rollback Procedure

### Quick Rollback (< 2 minutes)

```bash
# Step 1: Stop current version
docker stop basset-hound-browser
docker rm basset-hound-browser

# Step 2: Start previous version
docker run -d --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e DISPLAY=:99 \
  -e ELECTRON_DISABLE_SANDBOX=1 \
  --restart unless-stopped \
  basset-hound-browser:v12.0.0

# Step 3: Verify
docker logs basset-hound-browser
curl -s http://localhost:8765 | head -c 50
```

### Clean Rollback (Full Reset)

```bash
# Step 1: Stop and remove current container
docker stop basset-hound-browser
docker rm basset-hound-browser

# Step 2: Optional: Remove current image
docker rmi basset-hound-browser:v12.1.0

# Step 3: Start previous version
docker run -d --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e DISPLAY=:99 \
  -e ELECTRON_DISABLE_SANDBOX=1 \
  --restart unless-stopped \
  basset-hound-browser:v12.0.0

# Step 4: Restore database (if applicable)
# mongorestore --archive=backup.archive
# OR
# psql < backup.sql

# Step 5: Verify
sleep 10
curl -s http://localhost:8765 | head -c 50
```

### Rollback Verification Checklist

- [ ] Container started successfully
- [ ] No error messages in logs
- [ ] WebSocket server responding
- [ ] Health checks passing
- [ ] Baseline performance metrics met
- [ ] All critical features working

## Support & Escalation

### Getting Help

1. **Check Logs:**
   ```bash
   docker logs basset-hound-browser
   ```

2. **Review Documentation:**
   - API Reference: `docs/api/`
   - Feature Guides: `docs/features/`
   - Integration Guides: `docs/integration/`

3. **Run Diagnostics:**
   ```bash
   docker stats basset-hound-browser
   docker inspect basset-hound-browser
   docker network inspect basset-hound-browser
   ```

4. **Create Issue Report:**
   - Container logs (last 100 lines)
   - System metrics (RAM, CPU, disk)
   - Steps to reproduce
   - Expected vs actual behavior

### Emergency Procedures

1. **Container Hangs:** `docker restart basset-hound-browser`
2. **Port Conflict:** `lsof -i :8765` then kill/restart
3. **Memory Leak:** `docker restart basset-hound-browser`
4. **Data Corruption:** Rollback to previous version

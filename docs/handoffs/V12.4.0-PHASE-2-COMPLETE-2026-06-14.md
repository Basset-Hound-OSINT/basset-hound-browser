## Module 5: WebSocket Commands Integration

**File:** `websocket/commands/fake-data-commands.js` (210 LOC)

### Purpose
Provides WebSocket API commands for generating and managing fake data per session/tab.

### Commands (8 total)

#### 1. `generate_user_agent`
Generate spoofed user agent matching profile.
```json
{ "command": "generate_user_agent", "params": { "profile": {...}, "tabId": "tab1" } }
```
Response: `{ success: true, userAgent: "Mozilla/5.0...", validated: true }`

#### 2. `generate_screen_resolution`
Generate spoofed resolution matching profile.
```json
{ "command": "generate_screen_resolution", "params": { "profile": {...}, "tabId": "tab1" } }
```
Response: `{ success: true, resolution: { width: 390, height: 844, dpi: 2.0, ... }, validated: true }`

#### 3. `generate_gpu_specs`
Generate spoofed GPU/CPU specs matching profile.
```json
{ "command": "generate_gpu_specs", "params": { "profile": {...}, "tabId": "tab1" } }
```
Response: `{ success: true, specs: { cpu: "Apple A17 Pro", gpu: "Apple A17 Pro GPU", ... }, validated: true }`

#### 4. `generate_browser_profile`
Generate complete browser profile (plugins, timezone, language).
```json
{ "command": "generate_browser_profile", "params": { "profile": {...}, "tabId": "tab1" } }
```
Response: `{ success: true, browserProfile: { timezone: "America/New_York", languages: [...], ... }, validated: true }`

#### 5. `generate_all_fake_data`
Generate all data types at once with validation.
```json
{ "command": "generate_all_fake_data", "params": { "profile": {...}, "tabId": "tab1" } }
```
Response: `{ success: true, data: { userAgent: "...", resolution: {...}, specs: {...}, browserProfile: {...} }, validations: {...} }`

#### 6. `get_profile_consistency`
Verify all generated data is consistent within session.
```json
{ "command": "get_profile_consistency", "params": { "tabId": "tab1" } }
```
Response: `{ success: true, consistency: { allInitialized: true, hasUserAgent: true, ... } }`

#### 7. `reset_fake_data`
Reset all generated data for a tab.
```json
{ "command": "reset_fake_data", "params": { "tabId": "tab1" } }
```
Response: `{ success: true, message: "Fake data reset for tab tab1" }`

#### 8. `cleanup_tab`
Clean up resources for closed tab.
```json
{ "command": "cleanup_tab", "params": { "tabId": "tab1" } }
```
Response: `{ success: true, message: "Cleaned up resources for tab tab1" }`

#### Key Features
- **Per-Tab Generators:** Each tab maintains separate generator instances for consistency
- **Data Consistency:** Once generated, values remain consistent for tab lifetime
- **Comprehensive Validation:** All generated data validated before return
- **Tab Isolation:** No data leakage between tabs
- **Resource Cleanup:** Generators destroyed when tab closes

## Session 2: Data Directory Pattern

### Context
User requested standardized data directory pattern based on ResearchHub project.

### Research
Used parallel agents to review:
1. **BluePlan** - PDF ingestion, chunk size optimization
2. **ResearchHub** - Data directory structure pattern
3. **RAG Bootstrap** - Current state analysis

### Key Findings from BluePlan
- 512 token chunks caused 82% failure rate with Ollama
- Reduced to 150 tokens with 30 overlap for reliability
- Rate limiting (0.3s between chunks, 2.0s between files)
- MarkItDown recommended for multi-format document support

### Key Findings from ResearchHub
Data directory pattern:
```
data/
├── docker/          # Service data (postgres, redis)
├── cache/           # Embedding caches
├── logs/            # Application logs
├── exports/         # Generated outputs
└── registry/        # Metadata registries
```

Pattern uses bind mounts instead of Docker named volumes for visibility.

### Changes Made

1. **docker-compose.yml**
   - Added init service for directory creation
   - Changed from named volumes (pgdata, redisdata) to bind mounts (./data/docker/...)
   - Added depends_on for init service

2. **.gitignore** (NEW)
   - Comprehensive gitignore created
   - data/ excluded from version control
   - .env excluded
   - Python, IDE, OS patterns included

3. **deploy.sh**
   - Added `ensure_data_directories()` function
   - Enhanced `cmd_clean` to remove data directory contents

4. **config.yaml**
   - Reduced default chunk_size from 512 to 256 tokens
   - Added documentation about chunk size recommendations

## Data Directory Structure

```

├── docker/
│   ├── postgres/     # PostgreSQL database files
│   └── redis/        # Redis AOF persistence
├── cache/
│   └── embeddings/   # Embedding vector caches
├── logs/             # Application logs
├── exports/          # Generated exports
└── registry/         # Metadata registries
```

## Developer Experience

To customize for a new project:
1. Change `network.name` in config.yaml
2. Change `network.port` in config.yaml
3. Configure `ingestion.directories` in config.yaml
4. Run `ollama pull nomic-embed-text`
5. Run `./deploy.sh start --build`

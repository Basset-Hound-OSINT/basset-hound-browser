## Architecture

### Command Execution Flow

```
Client Request
  ↓
[Authentication & Rate Limiting]
  ↓
[ReliabilityManager.execute()]
  ├→ Command Execution (with timeout)
  ├→ Monitor for Errors
  ├→ Classify as Transient/Permanent
  ├→ Retry if Transient (max 3 times)
  ├→ Exponential Backoff (1s, 2s, 4s)
  └→ Record Metrics
  ↓
[HealthEndpoint.recordCommand()]
  ├→ Update per-command stats
  ├→ Calculate latency percentiles
  └→ Update global metrics
  ↓
Client Response (with reliability context)
```

### Retry Strategy

**Retryable Conditions**:
1. Error must be transient (TIMEOUT, ECONNRESET, ENOTFOUND, etc.)
2. Command must be retryable (GET operations, status queries, screenshots)

**Non-Retried**:
- Permanent errors (AUTH_FAILED, INVALID_PARAMETERS, etc.)
- Write operations
- Unknown commands

**Backoff Formula**: `delay = 1000 * (2 ^ attempt)`
- Attempt 1: Immediate
- Attempt 2: 1000ms
- Attempt 3: 2000ms
- Attempt 4: 4000ms (max attempts reached)

### Timeout Strategy

```
Standard:           30 seconds (30,000ms)
Large Response:     45 seconds (45,000ms) for HTML >5MB
Very Large:        120 seconds (120,000ms) for documents >20MB
```

## Protocol Overview

### Supported Protocols

| Protocol | URL Format | Use Case | Encryption |
|----------|-----------|----------|-----------|
| WebSocket | `ws://localhost:8765` | Real-time commands | Optional |
| WebSocket Secure | `wss://localhost:8765` | Production environments | TLS/SSL |
| HTTP REST | `http://localhost:8080` | Health/diagnostics | Optional |
| HTTPS REST | `https://localhost:8080` | Production health checks | TLS/SSL |

### Connection Lifecycle

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Establish WS Connection (with auth)                      │
│ 2. Send Ping (validate connection health)                   │
│ 3. Send Commands (execute browser automation)               │
│ 4. Handle Events (asynchronous notifications)               │
│ 5. Monitor Health (periodic status checks)                  │
│ 6. Close Connection (graceful shutdown)                     │
└─────────────────────────────────────────────────────────────┘
```

## Connection & Authentication

### WebSocket Connection Methods

#### Method 1: Query Parameter Authentication

```javascript
const token = 'your_authentication_token';
const ws = new WebSocket(`ws://localhost:8765?token=${token}`);

ws.onopen = () => {
  console.log('Connected and authenticated');
};
```

#### Method 2: Bearer Token Header

```javascript
const token = 'your_authentication_token';
const ws = new WebSocket('ws://localhost:8765', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

#### Method 3: Explicit Authenticate Command

```json
{
  "id": 1,
  "command": "authenticate",
  "token": "your_authentication_token"
}
```

**Response:**

```json
{
  "id": 1,
  "command": "authenticate",
  "success": true,
  "data": {

    "expiresAt": 1751673600000,
    "user": "default",
    "permissions": ["read", "write", "execute"]
  }
}
```

## Message Format & Versioning

### Standard Command Format

All WebSocket commands follow this structure:

```json
{
  "id": 12345,
  "command": "command_name",
  "version": "12.9.0",
  "priority": "normal",
  "timeout": 30000,
  "retry": {
    "enabled": true,
    "maxAttempts": 3,
    "backoffMs": 1000
  },
  "params": {
    "param1": "value1",
    "param2": "value2"
  }
}
```

### Standard Response Format

```json
{
  "id": 12345,
  "command": "command_name",
  "success": true,
  "duration": 145,
  "version": "12.9.0",
  "timestamp": 1683820982000,
  "data": {
    "result1": "value1",
    "result2": "value2"
  },
  "metadata": {
    "cached": false,
    "attempts": 1,
    "retried": false
  }
}
```

### Error Response Format

```json
{
  "id": 12345,
  "command": "command_name",
  "success": false,
  "error": {
    "code": "COMMAND_TIMEOUT",
    "message": "Command execution exceeded timeout of 30000ms",
    "details": {
      "command": "navigate",
      "timeout": 30000,
      "elapsed": 31200
    }
  },
  "recovery": {
    "retryable": true,
    "suggestedDelay": 1000,
    "nextAction": "Try again with increased timeout"
  }
}
```

### Versioning Strategy

- **API Version**: Sent in all requests (`version: "12.9.0"`)
- **Backward Compatibility**: v12.9.0 maintains full compatibility with v12.8.x
- **Deprecation Window**: 2 major versions (minimum 12 months)
- **Feature Flags**: New features behind flags for gradual rollout

## Reliability & SLA Management

### Overview

v12.9.0 introduces comprehensive reliability guarantees with automatic retry logic, SLA tracking, and health monitoring.

### Reliability Configuration

**Default SLA Targets:**

| Command Category | Target | Response Time P99 |
|---|---|---|
| Navigation | 99%+ | <2000ms |
| Extraction | 99%+ | <1000ms |
| Screenshots | 98%+ | <3000ms |
| All Commands | 95%+ | <2000ms |

### Automatic Retry Behavior

**Transient Errors (Automatically Retried):**
- ETIMEDOUT
- ECONNRESET
- ECONNREFUSED
- Temporary network failures

**Permanent Errors (Not Retried):**
- INVALID_PARAMETERS
- AUTH_FAILED
- UNAUTHORIZED
- Command not found

**Retry Configuration:**

```json
{
  "id": 40,
```

#### Strategy 2: Manual Retry with Backoff

```javascript
async function executeWithBackoff(command, maxAttempts = 3) {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      return await client.executeCommand(command);
    } catch (error) {
      if (!error.recovery.retryable || attempt === maxAttempts) {
        throw error;
      }
      const delay = error.recovery.suggestedDelay || (1000 * Math.pow(2, attempt - 1));
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
```

#### Strategy 3: Circuit Breaker

```javascript
class CircuitBreaker {
  constructor(threshold = 5, timeout = 60000) {
    this.failureCount = 0;
    this.successCount = 0;
    this.threshold = threshold;
    this.timeout = timeout;
    this.state = 'closed'; // closed, open, half-open
  }

  async execute(command) {
    if (this.state === 'open') {
      throw new Error('Circuit breaker is open');
    }

    try {
      const result = await client.executeCommand(command);
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }

  onSuccess() {
    this.failureCount = 0;
    if (this.state === 'half-open') {
      this.state = 'closed';
    }
  }

  onFailure() {
    this.failureCount++;
    if (this.failureCount >= this.threshold) {
      this.state = 'open';
      setTimeout(() => {
        this.state = 'half-open';
        this.failureCount = 0;
      }, this.timeout);
    }
  }
}
```

**Symptoms:** WebSocket connection never establishes

**Solutions:**
1. Increase timeout parameter: `"timeout": 60000` for slow networks
2. Check server load: `GET /health/detailed`
3. Use proxy with lower latency
4. Reduce parallel command count

#### Issue: High Memory Usage

**Symptoms:** Server process consumes excessive memory

**Solutions:**
1. Clear session data: `cleanup_expired_sessions`
2. Reduce screenshot cache: `cleanup_screenshot_manager`
3. Enable garbage collection: `curl -X POST http://localhost:8080/gc`
4. Reduce concurrent connections

**Symptoms:** AUTH_FAILED or MISSING_AUTH_TOKEN errors

**Solutions:**
1. Verify token format is correct

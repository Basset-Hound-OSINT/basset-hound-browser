## Incident Response Procedures

### Quick Reference: Incident Severity Levels

| Severity | Impact | Response Time | Escalation |
|----------|--------|----------------|------------|
| **CRITICAL** (P1) | Complete outage, 0% availability | 5 minutes | Immediate |
| **HIGH** (P2) | Severe degradation, >50% error rate | 15 minutes | Urgent |
| **MEDIUM** (P3) | Noticeable issues, 1-50% error rate | 30 minutes | Standard |
| **LOW** (P4) | Minimal impact, <1% error rate | 2 hours | Routine |

## Scenario 1: CRITICAL - Complete Service Outage

**Symptoms:**
- No response from WebSocket API (port 8765)
- All health checks failing
- 100% request error rate
- Unable to establish connections
- Client connection timeouts

**Detection:**
```bash
# Check service status
curl -s http://localhost:8765/health | jq '.status'
# Expected: "healthy" (if CRITICAL: error or timeout)

# Check container status
docker ps | grep basset-hound
# Expected: "Up X minutes" (if CRITICAL: not in list or Exited)
```

**Root Cause Analysis - Decision Tree:**

```
Service Down?
├─ Container not running? → START CONTAINER
├─ Port not accessible? → CHECK FIREWALL
├─ Process crashed? → CHECK LOGS
└─ All else → FULL RESTART
```

### Immediate Response (0-2 minutes)

**Step 1: Declare Incident**
```bash
# Start incident tracking
echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) - CRITICAL: Service Outage - Starting response" | tee -a /var/log/basset-incidents.log

# Notify on-call team
# Send Slack: @incidents - CRITICAL: Basset Hound Browser offline
```

**Step 2: Check Container Status**
```bash
# Check if container exists and is running
docker ps -a | grep basset-hound-browser

# Capture current state
docker inspect basset-hound-browser > /tmp/container-state.json

# Check Docker logs
docker logs basset-hound-browser 2>&1 | tail -100 > /tmp/docker-logs.txt
```

**Step 3: Quick Service Restart** (60 seconds)
```bash
# Option A: Restart existing container (if running)
docker restart basset-hound-browser
sleep 5

# Verify recovery
curl -s http://localhost:8765/health/live | jq '.'

# If still down, proceed to full restart
```

**Step 4: Full Container Restart** (if Step 3 failed)
```bash
# Stop container gracefully
docker stop -t 30 basset-hound-browser

# Remove container
docker rm basset-hound-browser

# Start fresh container
docker run -d \
  --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e NODE_ENV=production \
  -m 2g \
  --memory-swap 2g \
  -e LOG_LEVEL=info \
  basset-hound-browser:12.9.0

# Wait for startup
sleep 10

# Verify health
curl -s http://localhost:8765/health | jq '.status'
# Expected: "healthy"
```

**Step 5: Validate Service Recovery** (90 seconds total)
```bash
# Quick connectivity test
curl -s -X POST http://localhost:8765/health \
  -H "Content-Type: application/json" \
  -d '{"command":"ping"}' | jq '.success'
# Expected: true

# Check metrics
curl -s http://localhost:8765/health/metrics | jq '.requestCount'

# Send test through WebSocket
python3 << 'PYTHON'
import asyncio, websockets, json

async def test():
    async with websockets.connect("ws://localhost:8765") as ws:
        await ws.send(json.dumps({"command": "ping", "id": "test"}))
        response = await ws.recv()
        print("Response:", response)
        return "pong" in response.lower()

result = asyncio.run(test())
print("TEST PASSED" if result else "TEST FAILED")
PYTHON
```

### Escalation Path (if not recovered after Step 5)


**Step 9: Rollback Previous Deployment** (if recent deployment)

```bash
# Rollback to previous version
```

## Scenario 2: HIGH - Memory Exhaustion (OOM)

**Symptoms:**
- Memory usage trending toward 1900MB (90% of 2GB limit)
- Response times increasing
- WebSocket connections dropping
- `ENOMEM` errors in logs
- Container OOM killed events

**Detection:**
```bash
# Check memory usage
docker stats basset-hound-browser --no-stream | grep -E "MEM|basset"

# Monitor memory trend
watch -n 5 'docker stats basset-hound-browser --no-stream | tail -1'

# Check OOM events
docker inspect basset-hound-browser | jq '.State.OOMKilled'
# Expected: false (if true, container was killed for OOM)
```

### Immediate Response (0-5 minutes)

**Step 1: Assess Current State**
```bash
# Get exact memory usage
MEMORY=$(docker stats basset-hound-browser --no-stream | tail -1 | awk '{print $(NF-2)}')
echo "Current Memory: $MEMORY"

# Get limit
LIMIT=$(docker inspect basset-hound-browser | jq '.HostConfig.Memory / 1073741824' | head -1)
echo "Memory Limit: ${LIMIT}GB"

# Calculate percentage
PERCENT=$(docker stats basset-hound-browser --no-stream | tail -1 | grep -oP '\d+(?=%)' | head -1)
echo "Memory Percent: $PERCENT%"
```

**Step 2: Choose Response Based on Percentage**

**If 75-85% (Yellow Alert):**
```bash
# Monitor closely but don't act yet
# Set up continuous monitoring
for i in {1..12}; do
  echo "$(date): $(docker stats basset-hound-browser --no-stream | tail -1 | awk '{print $(NF-2)}')"
  sleep 5
done > /tmp/memory-trend.txt

# If trending down, continue monitoring
# If trending up, proceed to 85%+ steps
```

**If 85-95% (Orange Alert):**
```bash
# Grace shutdown of idle connections
curl -s -X POST http://localhost:8765/api/shutdown-idle \
  -H "Content-Type: application/json" \
  -d '{"inactiveMs": 30000}'

# Trigger garbage collection
curl -s http://localhost:8765/api/gc \
  -H "Content-Type: application/json" | jq '.'

# Wait and check
sleep 10
docker stats basset-hound-browser --no-stream
```

**If >95% or OOMKilled=true (Red Alert):**
```bash
# Immediate graceful restart
docker stop -t 30 basset-hound-browser

# Restart with increased memory limit (temporary)
docker run -d \
  --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -m 3g \
  --memory-swap 3g \
  --memory-reservation 2.5g \
  -e NODE_ENV=production \
  basset-hound-browser:12.9.0

# Monitor after restart
watch -n 5 'docker stats basset-hound-browser --no-stream | tail -1'
```

### Root Cause Investigation (5-15 minutes)

**Step 3: Identify Memory Leak**
```bash
# Get heap statistics
curl -s http://localhost:8765/health/metrics | jq '.memory' > /tmp/heap-stats.json

# Check if memory is growing
echo "Memory from health endpoint:"
for i in {1..5}; do
  curl -s http://localhost:8765/health/metrics | jq '.memory'
  sleep 30
done | tee /tmp/memory-samples.json

# Analyze growth
python3 << 'PYTHON'
import json, time

samples = []
for i in range(5):
    import subprocess
    result = subprocess.run(
        'curl -s http://localhost:8765/health/metrics | jq ".memory.heapUsedMB"',
        shell=True, capture_output=True, text=True
    )
    if result.stdout.strip():
        samples.append(float(result.stdout.strip()))
    time.sleep(30)

if len(samples) > 1:
    growth_rate = (samples[-1] - samples[0]) / 4  # per minute
    print(f"Memory growth rate: {growth_rate:.2f}MB/min")
    if growth_rate > 5:
        print("WARNING: Significant memory leak detected!")
PYTHON
```

**Step 4: Check for Common Leak Sources**
```bash
# High WebSocket connections
curl -s http://localhost:8765/health/metrics | jq '.websocket.activeConnections'

# Long-running requests
curl -s http://localhost:8765/health/metrics | jq '.requests.longest'

# Screenshot cache size
docker exec basset-hound-browser du -sh /tmp/screenshots/ 2>/dev/null || echo "Screenshot dir not found"

# Check for orphaned processes
docker top basset-hound-browser | wc -l

# Query process memory
docker exec basset-hound-browser ps aux --sort=-%mem | head -10
```

**Step 5: Determine Action**

**Memory growing < 5MB/min:**
- Likely normal behavior
- Monitor for 1 hour
- Continue if stable
- No action needed

**Memory growing 5-50MB/min:**
- Likely slow leak
- Identify source (connections, cache, etc.)
- Update scaling policy
- Schedule for next maintenance window

**Memory growing > 50MB/min:**
- Critical leak
- Immediate restart required
- CRITICAL incident escalation
- Revert to previous version

### Resolution Options

**Option A: Connection Cleanup** (lowest risk)
```bash
# Close idle connections (30s+ inactive)
curl -s -X POST http://localhost:8765/api/connections/cleanup \
  -H "Content-Type: application/json" \
  -d '{
    "idleThresholdMs": 30000,
    "forceClose": false
  }' | jq '.'

# Verify
sleep 5
curl -s http://localhost:8765/health/metrics | jq '.websocket.activeConnections'
```

**Option B: Cache Eviction** (low-medium risk)
```bash
# Clear screenshot cache (if not needed for active sessions)
curl -s -X DELETE http://localhost:8765/api/cache/screenshots \
  -H "Content-Type: application/json" | jq '.'

# Check freed space
docker exec basset-hound-browser du -sh /tmp/screenshots/
```

**Option C: Graceful Restart** (medium risk)
```bash
# Enable drain mode (stop accepting new connections)
curl -s -X POST http://localhost:8765/api/drain \
  -H "Content-Type: application/json" | jq '.'

# Wait for existing connections to close (max 2 minutes)
sleep 120

# Force close remaining connections
curl -s -X POST http://localhost:8765/api/shutdown \
  -H "Content-Type: application/json" \
  -d '{"gracePeriodMs": 5000}' | jq '.'

# Container will restart automatically (if using restart policy)
docker wait basset-hound-browser  # Wait for exit
sleep 5

# Verify restart
docker ps | grep basset-hound
```

**Option D: Version Upgrade** (if upgrade available)
```bash
# Deploy newer version with fixes
scripts/deploy-v12.9.0.sh --target-version 12.10.0 --canary

# Monitor memory
watch -n 5 'docker stats basset-hound-browser --no-stream | tail -1'
```

## Scenario 3: MEDIUM - High Error Rate Surge

**Symptoms:**
- Error rate jumps to 5-50% (normally <1%)
- Specific commands failing (e.g., `navigate`, `screenshot`)
- Client reports failures intermittently
- Health check shows degraded SLA compliance
- Latency increases significantly

**Detection:**
```bash
# Check error rate
curl -s http://localhost:8765/health/metrics | jq '.errorRate'
# Expected: <1% (if MEDIUM: 5-50%)

# Check per-command errors
curl -s http://localhost:8765/health/metrics | jq '.topCommands[] | select(.errors > 0)'

# Check SLA compliance
curl -s http://localhost:8765/health/reliability | jq '.sla'
```

### Immediate Response (0-5 minutes)

**Step 1: Identify Affected Command**
```bash
# Get detailed error breakdown
curl -s http://localhost:8765/health/metrics | jq '
  .topCommands |
  map(select(.errors > 0)) |
  sort_by(-.errors) |
  .[0:5]
' > /tmp/error-breakdown.json

# Show results
jq '.' /tmp/error-breakdown.json
```

**Step 2: Check Server Logs for Error Patterns**
```bash
# Get last 200 error log entries
docker logs basset-hound-browser 2>&1 | grep -i "error\|exception\|fail" | tail -200 > /tmp/error-logs.txt

# Analyze error types
grep -oP 'Error: \K[^[:space:]]+' /tmp/error-logs.txt | sort | uniq -c | sort -rn

# Sample specific error
docker logs basset-hound-browser 2>&1 | grep -i "error" | head -1
```

**Step 3: Determine Error Category**

**Category A: Network/Timeout Errors**
```
Pattern: ETIMEDOUT, ECONNRESET, TIMEOUT
Solution: → See Step 4A
```

**Category B: Resource Exhaustion**
```
Pattern: ENOMEM, ENOBUFS, out of memory
Solution: → See Step 4B (Memory Exhaustion scenario)
```

**Category C: Specific Command Failures**
```
Pattern: navigate, screenshot, click failures
Solution: → See Step 4C
```

**Category D: External Service Errors**
```
Pattern: Connection to external service failed
Solution: → See Step 4D
```

### Step 4: Targeted Resolution

**Step 4A: Network/Timeout Errors**
```bash
# Check network connectivity
docker exec basset-hound-browser ping -c 5 8.8.8.8

```

**Step 4C: Specific Command Failures**
```bash

# Test command
python3 << 'PYTHON'

async def test_navigate():

        print(f"Success: {response.get('success')}")

```

## Troubleshooting Guide

### Common Issues & Solutions

#### Issue 1: WebSocket Connection Refused

**Error Message:**
```
Error: ECONNREFUSED - Connection refused 127.0.0.1:8765
```

**Diagnostic Steps:**
```bash
# 1. Check container is running
docker ps | grep basset-hound

# 2. Check port is listening
netstat -tuln | grep 8765

# 3. Test port accessibility
curl -s http://localhost:8765/health

# 4. Check firewall
sudo ufw status | grep 8765

# 5. Check for port conflicts
lsof -i :8765
```

**Solutions:**
| Cause | Solution |
|-------|----------|
| Container not running | `docker start basset-hound-browser` |
| Port in use | `lsof -i :8765` → `kill -9 <PID>` |
| Firewall blocking | `sudo ufw allow 8765` |
| Wrong hostname/IP | Use `127.0.0.1` or container IP |

---

#### Issue 2: WebSocket Connection Timeout

**Error Message:**
```
WebSocketTimeoutError: Connection timed out after 30s
```

**Diagnostic Steps:**

#### Scaling from 1 to 3 Instances

**Architecture:**
```
Client Connections
       │
       ▼
   HAProxy (Load Balancer, Port 8765)
       │
    ┌──┼──┐
    ▼  ▼  ▼
   Node1 Node2 Node3  (Basset Hound instances)
 :9001 :9002 :9003    (Internal ports)
```

**Step 1: Create Additional Instances**
```bash
#!/bin/bash

# Instance configuration
INSTANCES=3
BASE_PORT=9001
NETWORK="basset-hound-browser"
IMAGE="basset-hound-browser:12.9.0"
MEMORY="2g"
CPUS="2"

for i in $(seq 1 $INSTANCES); do
  PORT=$((BASE_PORT + i - 1))
  CONTAINER_NAME="basset-hound-browser-$i"
  
  echo "Starting $CONTAINER_NAME on port $PORT..."
  
  docker run -d \
    --name "$CONTAINER_NAME" \
    --network "$NETWORK" \
    -p "$PORT:8765" \
    -e NODE_ENV=production \
    -e INSTANCE_ID="$i" \
    -m "$MEMORY" \
    --cpus="$CPUS" \
    "$IMAGE"
  
  # Wait for startup
  sleep 5
  
  # Verify health
  if curl -s http://localhost:$PORT/health | jq -e '.status == "healthy"' > /dev/null; then
    echo "✓ $CONTAINER_NAME started successfully"
  else
    echo "✗ $CONTAINER_NAME failed to start"
    docker logs "$CONTAINER_NAME" | tail -20
    exit 1
  fi
done

echo ""
echo "All instances started. Configure load balancer to balance across:"
for i in $(seq 1 $INSTANCES); do
  PORT=$((BASE_PORT + i - 1))
  echo "  - 127.0.0.1:$PORT"
done
```

**Step 2: Configure Load Balancer**

**HAProxy Configuration** (`/etc/haproxy/haproxy.cfg`):
```haproxy
global
  maxconn 4000
  log 127.0.0.1 local0 info

defaults
  mode tcp
  timeout connect 5000ms
  timeout client 50000ms
  timeout server 50000ms
  log global
  option tcplog

# WebSocket frontend (port 8765)
frontend websocket_in
  bind *:8765
  mode tcp
  default_backend websocket_backends
  
  # Logging
  option tcplog
  capture request header User-Agent len 64

# Backend servers
backend websocket_backends
  balance roundrobin
  mode tcp
  
  # Health checks
  option tcp-check
  tcp-check connect port 8765
  
  # Servers
  server node1 127.0.0.1:9001 check inter 5000 fall 2 rise 2
  server node2 127.0.0.1:9002 check inter 5000 fall 2 rise 2
  server node3 127.0.0.1:9003 check inter 5000 fall 2 rise 2
  
  # Rate limiting per connection
  rate-limit sessions 1000

# Stats page
listen stats
  bind *:8404
  stats enable
  stats uri /haproxy?stats
  stats refresh 30s
```

## Backup & Restore Procedures

### Backup Strategy

**What to Backup:**
1. **Container State** (profiles, fingerprints, session data)
2. **Configuration** (env vars, settings, secrets)
3. **Logs** (for debugging, audit trail)
4. **Database** (if using external database)

**What NOT to Backup:**
- Large cache files (screenshots, temporary downloads)
- npm node_modules (can be regenerated)
- Docker image layers (stored in registry)

#### Backup Types

**Type 1: Daily Container State Backup**

```bash
#!/bin/bash
# backup-container-state.sh

BACKUP_DIR="/backups/basset-hound"
TIMESTAMP=$(date +%Y-%m-%d_%H-%M-%S)
BACKUP_FILE="$BACKUP_DIR/state-$TIMESTAMP.tar.gz"

mkdir -p "$BACKUP_DIR"

# Stop container gracefully
docker stop -t 60 basset-hound-browser

# Copy container volumes
docker cp basset-hound-browser:/home/app/state /tmp/basset-state-$TIMESTAMP
docker cp basset-hound-browser:/home/app/profiles /tmp/basset-profiles-$TIMESTAMP

# Restart container
docker start basset-hound-browser

# Compress backup
tar -czf "$BACKUP_FILE" -C /tmp basset-state-$TIMESTAMP basset-profiles-$TIMESTAMP

# Cleanup
rm -rf /tmp/basset-state-$TIMESTAMP /tmp/basset-profiles-$TIMESTAMP

# Verify backup
if tar -tzf "$BACKUP_FILE" > /dev/null 2>&1; then
  echo "Backup successful: $BACKUP_FILE"
  echo "Size: $(du -h $BACKUP_FILE | cut -f1)"
else
  echo "ERROR: Backup verification failed"
  rm "$BACKUP_FILE"
  exit 1
fi

# Upload to remote storage (S3 example)
aws s3 cp "$BACKUP_FILE" "s3://basset-hound-backups/daily/$TIMESTAMP.tar.gz"

# Cleanup old backups (keep last 30 days)
find "$BACKUP_DIR" -name "state-*.tar.gz" -mtime +30 -delete
```


### Restore Procedures

#### Scenario A: Single-File Restore

**Restore container state from backup:**
```bash
#!/bin/bash
# restore-state.sh

if [ -z "$1" ]; then
  echo "Usage: $0 <backup-file>"
  echo ""
  echo "Available backups:"
  ls -lh /backups/basset-hound/state-*.tar.gz | tail -10
  exit 1
fi

BACKUP_FILE="$1"
RESTORE_DIR="/tmp/restore-$$"

echo "Restoring from: $BACKUP_FILE"

# Verify backup integrity
if ! tar -tzf "$BACKUP_FILE" > /dev/null; then
  echo "ERROR: Backup file is corrupted"
  exit 1
fi

# Create restore directory
mkdir -p "$RESTORE_DIR"

# Extract backup
tar -xzf "$BACKUP_FILE" -C "$RESTORE_DIR"

# Stop container
echo "Stopping container..."
docker stop -t 60 basset-hound-browser

# Clear existing state
docker exec basset-hound-browser rm -rf /home/app/state/* /home/app/profiles/* 2>/dev/null || true

# Restore state
echo "Restoring state..."
docker cp "$RESTORE_DIR/basset-state-"*"/" basset-hound-browser:/home/app/state/ 2>/dev/null || true
docker cp "$RESTORE_DIR/basset-profiles-"*"/" basset-hound-browser:/home/app/profiles/ 2>/dev/null || true

# Start container
```

### Disaster Recovery

#### Total Data Loss Scenario

**Situation:** All container data lost, need to recover from backup

```bash
#!/bin/bash
# dr-restore-full.sh

set -e

# Configuration
BACKUP_FILE="${1:-latest}"  # "latest" or specific file path
RESTORE_TIMESTAMP=$(date +%Y-%m-%d_%H-%M-%S)

echo "=== Full Disaster Recovery Restore ==="
echo "Starting: $RESTORE_TIMESTAMP"

# Step 1: Find backup
if [ "$BACKUP_FILE" == "latest" ]; then
  BACKUP_FILE=$(ls -t /backups/basset-hound/state-*.tar.gz 2>/dev/null | head -1)
  if [ -z "$BACKUP_FILE" ]; then
    echo "ERROR: No backup files found"
    exit 1
  fi
fi

echo "Using backup: $BACKUP_FILE"
echo "Backup age: $(date -r $BACKUP_FILE)"

# Step 2: Verify backup
echo "Verifying backup integrity..."
if ! tar -tzf "$BACKUP_FILE" > /dev/null; then
  echo "ERROR: Backup is corrupted, cannot restore"
  exit 1
fi

# Step 3: Clean slate
echo "Removing existing container..."
docker stop basset-hound-browser 2>/dev/null || true
docker rm basset-hound-browser 2>/dev/null || true
sleep 2

# Step 4: Create fresh container
echo "Creating fresh container..."
docker run -d \
  --name basset-hound-browser \
  --network basset-hound-browser \
  -p 8765:8765 \
  -v basset-hound-data:/home/app/state \
  -v basset-hound-profiles:/home/app/profiles \
  -e NODE_ENV=production \
  basset-hound-browser:12.9.0

sleep 5

# Step 5: Restore data
echo "Restoring data from backup..."
RESTORE_DIR="/tmp/restore-$$"
mkdir -p "$RESTORE_DIR"
tar -xzf "$BACKUP_FILE" -C "$RESTORE_DIR"

# Copy to container volumes
docker cp "$RESTORE_DIR/basset-state-"*"/." basset-hound-browser:/home/app/state/ 2>/dev/null || true
docker cp "$RESTORE_DIR/basset-profiles-"*"/." basset-hound-browser:/home/app/profiles/ 2>/dev/null || true

rm -rf "$RESTORE_DIR"

# Step 6: Verify
echo "Verifying recovery..."
sleep 5

ATTEMPTS=0
while [ $ATTEMPTS -lt 5 ]; do
  if curl -s http://localhost:8765/health | jq -e '.status == "healthy"' > /dev/null; then
    echo "SUCCESS: Container is healthy"
    curl -s http://localhost:8765/health | jq '{version, status, timestamp: .liveness.timestamp}'
    echo ""
    echo "Restore completed: $(date)"
    exit 0
  fi
  
  ATTEMPTS=$((ATTEMPTS + 1))
  echo "Waiting for container to start (attempt $ATTEMPTS/5)..."
  sleep 5
done

echo "ERROR: Container failed to become healthy"
docker logs basset-hound-browser | tail -50
exit 1
```

#### Multi-Region Recovery

**For distributed deployments, restore to secondary region:**

```bash
#!/bin/bash
# dr-restore-secondary-region.sh

PRIMARY_REGION="us-east-1"
SECONDARY_REGION="us-west-2"
BACKUP_BUCKET="s3://basset-hound-backups"

echo "Restoring to secondary region: $SECONDARY_REGION"

# Download latest backup from S3
echo "Downloading latest backup..."
LATEST_BACKUP=$(aws s3 ls "$BACKUP_BUCKET/daily/" --region $PRIMARY_REGION | \
  awk '{print $NF}' | sort | tail -1)

aws s3 cp \
  "$BACKUP_BUCKET/daily/$LATEST_BACKUP" \
  "/tmp/latest-backup.tar.gz" \
  --region $PRIMARY_REGION

# Restore to secondary container
docker run -d \
  --name basset-hound-browser-dr \
  --network basset-hound-browser \
  -p 8765:8765 \
  -e NODE_ENV=production \
  -e DR_MODE=true \
  basset-hound-browser:12.9.0

# Restore data
sleep 5
RESTORE_DIR="/tmp/restore-dr"
mkdir -p "$RESTORE_DIR"
tar -xzf /tmp/latest-backup.tar.gz -C "$RESTORE_DIR"
docker cp "$RESTORE_DIR"/*"/" basset-hound-browser-dr:/home/app/ 2>/dev/null || true

# Verify
sleep 5
curl -s http://localhost:8765/health | jq '.status'

echo "DR container running. Update DNS/routing to point to secondary region."
```

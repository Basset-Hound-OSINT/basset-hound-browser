### Compression Commands (5)

#### 1. `compress`
- **Purpose:** Compress single payload with adaptive algorithm selection
- **Parameters:** payload (Buffer/string), options (Object)
- **Returns:** { compressed (Buffer), algorithm (string), ratio (number), duration (number) }
- **Performance:** <10ms latency
- **Example:**
  ```javascript
  {
    "command": "compress",
    "payload": "large data string",
    "options": { "algorithm": "auto", "level": 6 }
  }
  ```

#### 2. `compressMultiple`
- **Purpose:** Compress multiple payloads efficiently
- **Parameters:** payloads (Array), options (Object)
- **Returns:** Array of { compressed, algorithm, ratio, duration }
- **Performance:** <20ms for 10 payloads
- **Example:**
  ```javascript
  {
    "command": "compressMultiple",
    "payloads": ["data1", "data2", "data3"],
    "options": { "algorithm": "auto" }
  }
  ```

#### 3. `getCompressionMetrics`
- **Purpose:** Get real-time compression metrics
- **Parameters:** None
- **Returns:** { algorithms: Map, totalOperations: number, averageRatio: number, ... }
- **Performance:** <1ms
- **Example:**
  ```javascript
  { "command": "getCompressionMetrics" }
  ```

#### 4. `getCompressionStatistics`
- **Purpose:** Get detailed compression statistics
- **Parameters:** timeWindow (number, optional)
- **Returns:** { statistics object with percentiles, trends, algorithms }
- **Performance:** <5ms
- **Example:**
  ```javascript
  {
    "command": "getCompressionStatistics",
    "timeWindow": 3600000
  }
  ```

#### 5. `resetCompressionMetrics`
- **Purpose:** Clear compression metrics

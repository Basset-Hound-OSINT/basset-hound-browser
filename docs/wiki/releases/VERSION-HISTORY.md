## Version Naming Convention

**Format:** `MAJOR.MINOR.PATCH`

- **MAJOR:** Significant new capabilities (e.g., multi-browser support)
- **MINOR:** Feature additions and improvements (current)
- **PATCH:** Bug fixes and stability improvements

**Examples:**
- 12.8.0 = Major feature set (forensic commands), minor 8, patch 0
- 13.0.0 = Major new architecture (multi-browser)
- 12.8.1 = Bug fix in 12.8 line

## Contributing to Version Control

When proposing changes:

1. Check current version in `package.json`
2. Determine if MAJOR, MINOR, or PATCH bump
3. Update version before creating pull request
4. Add entry to release notes
5. Run full test suite
6. Document breaking changes (if any)

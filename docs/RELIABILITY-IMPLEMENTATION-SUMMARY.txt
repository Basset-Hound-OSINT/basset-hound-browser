TECHNICAL SPECIFICATIONS
========================

Retry Strategy:
- Base Timeout: 30 seconds (adaptive: up to 120 seconds for large documents)
- Max Retries: 3 attempts
- Backoff: Exponential (1s, 2s, 4s between attempts)
- Total Max Time: ~7 seconds (immediate + 1s + 2s + 4s)

Transient Errors (Retried):
- ETIMEDOUT, ECONNRESET, ECONNREFUSED
- EPIPE, ENOTFOUND, ENETUNREACH
- EAI_AGAIN, TIMEOUT
- EADDRINUSE, EHOSTUNREACH, socket hang up

Permanent Errors (Not Retried):
- INVALID_PARAMETERS, AUTH_FAILED
- UNAUTHORIZED, FORBIDDEN, NOT_FOUND
- BAD_REQUEST, Unknown command

Retryable Commands:
- All GET operations (get_*, list_*)
- Screenshots (screenshot, screenshot_full_page, etc.)
- Status queries (status, ping, getHealth, etc.)
- NOT write operations (fill, click, navigate when idempotent)

Metrics Retention:
- In-Memory: Last 5000 requests
- Per-Command: All-time aggregation
- Latency Samples: Last 100 per command
- Global Stats: All-time tracking
- Lifetime: Server uptime or 24 hours

Performance:
- Latency Overhead: <1ms per command
- Memory Overhead: 1-2MB for typical operations
- CPU Overhead: <1% for metrics collection
- No throughput degradation

KEY REQUIREMENTS SUMMARY
================================================================================

FUNCTIONAL REQUIREMENTS (10 Total)
-----------------------------------

FR-FC-001: Full Page HTML Extraction
  - Capture rendered HTML in raw, prettified, minified formats
  - Include all DOM elements, Shadow DOM, iframes
  - Provide metrics (element count, size, tag types)

FR-FC-002: DOM Structure Snapshot
  - Hierarchical DOM tree with computed styles
  - Element visibility state and bounding boxes
  - XPath for element location
  - Form field metadata

FR-FC-003: Complete JavaScript Code Extraction
  - Inline scripts, external scripts, dynamic scripts
  - Script execution order and dependencies
  - Bundler detection (webpack, rollup, parcel)
  - Source maps and minification detection

FR-FC-004: JavaScript Execution Context
  - Window object state and global variables
  - Loaded libraries (jQuery, React, Angular, etc.)
  - Console logs, event listeners, timers
  - localStorage, sessionStorage, IndexedDB contents

FR-FC-005: Complete CSS Extraction
  - Inline styles, external stylesheets, @imports
  - CSS variables (custom properties)
  - Font imports and media queries
  - CSS-in-JS framework detection

FR-FC-006: Network Request/Response Capture
  - HTTP/HTTPS requests and responses (HAR format)
  - Headers, bodies, timing information
  - DNS, connect, SSL/TLS, wait, download timings
  - TLS certificate information
  - Redirect chains and blocked requests

FR-FC-007: TLS Certificate Extraction
  - Server certificate and full chain
  - Subject, issuer, validity dates
  - Public key, serial number, extensions
  - Cipher suite and protocol version

FR-FC-008: Multi-Format Export
  - JSON, CSV, XML, HTML formats
  - ZIP and TAR.GZ compression
  - SHA-256 integrity hashes
  - File manifest with metadata

FR-FC-009: Batch Operations
  - Multiple URL processing
  - Parallel and sequential execution modes
  - Progress tracking and callbacks
  - Partial failure handling

FR-FC-010: Encrypted Export
  - AES-256-GCM encryption
  - Password-based key derivation (PBKDF2)
  - Direct key provision option
  - Selective field encryption

API CONTRACT SPECIFICATION
================================================================================

Protocol: WebSocket (default port 8765)
Message Format: JSON

Command Families (8 total):
  1. HTML Capture (2 commands)
  2. JavaScript Capture (2 commands)
  3. CSS Capture (2 commands)
  4. Network Capture (3 commands)
  5. Export (3 commands)
  6. Batch (3 commands)
  7. Encryption (2 commands)
  8. Verification (3 commands)

Key Commands:
  - capture_html: Capture complete rendered HTML
  - capture_dom_snapshot: Capture DOM structure with metadata
  - capture_scripts: Extract all JavaScript code
  - capture_stylesheets: Extract all CSS styling
  - capture_network_traffic: Capture HTTP/HTTPS traffic (HAR)
  - export_forensic_data: Export in JSON/CSV/XML/HTML
  - batch_capture_forensic: Process multiple URLs
  - export_forensic_encrypted: Export with AES-256 encryption

CLIENT LIBRARIES
================================================================================

Python Client (basset_hound_client)
  from basset_hound_client import BrowserClient, ForensicCapture

  client = BrowserClient('ws://localhost:8765')
  capture = ForensicCapture(client)

  html_result = capture.html(url='https://example.com')
  batch = capture.batch_urls(urls=[...], types=['html', 'scripts'])

JavaScript/Node.js Client (basset-hound-client)
  const BrowserClient = require('basset-hound-client');

  const client = new BrowserClient('ws://localhost:8765');

  const html = await client.forensic.html({url: 'https://example.com'});
  const batch = await client.forensic.batchCapture({urls: [...], ...});

## Architecture & Design

### Integration Pattern: Modular Registration
Each feature is self-contained in a separate file with:
- Standardized `registerXxxCommands(commandHandlers)` interface
- Consistent error handling (`{success, error/data}` format)
- No shared state between features
- Independent initialization and cleanup

### Dependency Chain
```
Credentials
  → No external dependencies (uses Node crypto)
  
Session Persistence
  → BrowserStateCapture (src/sessions/state-capture.js)
  → BrowserStateRestore (src/sessions/state-restore.js)
  
Extended Evasion
  → Self-contained EvasionConfigurationManager
  → No external module dependencies
  
Monitoring & Metrics
  → Optional monitoring module (lazy-loaded)
  → Graceful fallback if module unavailable
```

### Error Handling
- All commands wrapped in try-catch blocks
- Consistent error response format
- Detailed error messages for debugging
- Graceful degradation for missing managers

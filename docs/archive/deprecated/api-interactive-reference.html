Connection Guide

WebSocket URLs

// Development (unencrypted) const ws = new WebSocket('ws://localhost:8765'); // Production (with TLS) const ws = new WebSocket('wss://localhost:8765'); // Docker network const ws = new WebSocket('ws://basset-hound-browser:8765');

Connection Lifecycle

const ws = new WebSocket('ws://localhost:8765'); // 1. Wait for connection ws.onopen = () => { console.log('Connected'); }; // 2. Send commands ws.send(JSON.stringify({ id: 'req-1', command: 'navigate', url: 'https://example.com' })); // 3. Receive responses ws.onmessage = (event) => { const response = JSON.parse(event.data); if (response.success) { console.log('Success:', response.data); } else { console.error('Error:', response.error); } }; // 4. Handle disconnection ws.onclose = () => { console.log('Disconnected'); };

Connection Timeout

Each request must complete within the specified timeout (default 30 seconds).

Authentication

Authentication is optional by default in development mode. Enable authentication in production environments.

Authentication Methods

Query Parameter

const token = 'YOUR_API_TOKEN'; const ws = new WebSocket(`ws://localhost:8765?token=${token}`);

HTTP Header

const options = { headers: { 'Authorization': 'Bearer YOUR_API_TOKEN' } }; const ws = new WebSocket('ws://localhost:8765', options);

Authenticate Command

// Send after connection ws.send(JSON.stringify({ id: 'auth-1', command: 'authenticate', token: 'YOUR_API_TOKEN' }));

Rate Limiting

The API uses per-command rate limiting with a sliding window algorithm to prevent abuse and ensure fair resource allocation.

Global Limits

Client Type Limit
Unauthenticated 100 req/min
Authenticated 1000 req/min

Per-Command Limits

Command Limit Reason
screenshot 5 req/min CPU/memory intensive
navigate 15 req/min Network overhead
execute_script 20 req/min Dangerous operation
get_content 100 req/min Safe read operation

Rate Limit Response

{ "success": false, "error": "Rate limit exceeded", "code": "RATE_LIMITED", "data": { "limit": 100, "window": "60000ms", "remaining": 0, "resetAt": 1687345678000, "retryAfter": 15000 } }

Check Rate Limit Status

Use get_rate_limit_status to check remaining requests (doesn't count toward limit):

// Request { "id": "status-1", "command": "get_rate_limit_status" } // Response { "success": true, "data": { "remaining": 47, "limit": 100, "window": "60000ms", "resetAt": 1687345678000 } }

Error Codes Reference

Standard Error Codes

4xx - Client Errors

400 BAD_REQUEST

Invalid request format or missing required parameters

401 UNAUTHORIZED

Authentication failed or token invalid

403 FORBIDDEN

Permission denied for this operation

404 NOT_FOUND

Resource not found

409 CONFLICT

Resource already exists or state conflict

413 SIZE_EXCEEDED

Request payload exceeds size limit

429 RATE_LIMITED

Rate limit exceeded, retry after delay

Command-Specific Error Codes

5xx - Server Errors

500 INTERNAL_ERROR

Unexpected server-side error

504 TIMEOUT

Command execution timeout exceeded

Error Response Format

{ "id": "req-1", "command": "screenshot", "success": false, "error": "Timeout waiting for page load", "code": "TIMEOUT", "details": { "timeout": 30000, "elapsed": 30234 }, "recovery": { "suggestion": "Increase timeout or simplify page", "alternativeCommands": [ "screenshot_viewport", "screenshot_element" ] } }

Code Examples

JavaScript Client

const BassetClient = require('basset-hound-client'); async function example() { const client = new BassetClient({ url: 'ws://localhost:8765', token: 'YOUR_TOKEN' }); await client.connect(); // Navigate await client.execute('navigate', { url: 'https://example.com' }); // Take screenshot const screenshot = await client.execute('screenshot', {}); console.log('Screenshot:', screenshot.data.imageData); // Capture evidence const evidence = await client.execute('capture_screenshot_evidence', { imageData: screenshot.data.imageData, url: 'https://example.com' }); console.log('Evidence ID:', evidence.evidenceId); await client.disconnect(); } example().catch(console.error);

Python Client

from basset_hound import BassetClient async def example(): client = BassetClient( url='ws://localhost:8765', token='YOUR_TOKEN' ) await client.connect() # Navigate await client.execute('navigate', url='https://example.com') # Take screenshot screenshot = await client.execute('screenshot') print(f'Screenshot size: {len(screenshot["data"]["imageData"])}') # Capture evidence evidence = await client.execute('capture_screenshot_evidence', imageData=screenshot['data']['imageData'], url='https://example.com' ) print(f'Evidence ID: {evidence["evidenceId"]}') await client.disconnect() # Run example import asyncio asyncio.run(example())

Best Practices

Connection Management

  • Use connection pooling for high-throughput applications
  • Implement automatic reconnection with exponential backoff
  • Monitor connection health with periodic ping commands
  • Set appropriate timeouts based on command type (5-60 seconds)
  • Use separate connections for parallel operations

Error Handling

  • Check success field in every response
  • Implement retry logic for transient errors (TIMEOUT, RATE_LIMITED)
  • Use exponential backoff with jitter for retries
  • Log error codes and details for debugging
  • Use recovery suggestions from error response

Rate Limiting

  • Check remaining quota with get_rate_limit_status
  • Batch heavy operations (screenshots, forensics capture)
  • Spread requests evenly across the rate limit window
  • Authenticate to increase limits from 100 to 1000 req/min
  • Respect retryAfter header when rate limited

Forensic Evidence

  • Always capture metadata (URL, title, timestamp)
  • Use provided evidence IDs for chain of custody
  • Verify hash integrity before using evidence
  • Package evidence with manifests for admissibility
  • Export in standard formats (JSON, CSV, WARC)

Performance Optimization

  • Use viewport screenshots instead of full-page when possible
  • Enable compression for large responses (gzip, brotli)
  • Batch multiple commands in parallel requests
  • Monitor memory usage and cleanup resources regularly
  • Use appropriate timeout values to avoid resource waste

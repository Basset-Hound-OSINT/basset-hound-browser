AREA 3: LOWER-LEVEL INTERACTION (28 Commands)
═════════════════════════════════════════════════════════════════════════════

1. Direct DOM Access (7 commands)
   ├── GET_ELEMENT_PROPERTY: Access element properties with type info
   ├── SET_ELEMENT_PROPERTY: Modify element properties safely
   ├── GET_COMPUTED_STYLES: Extract computed CSS for elements
   ├── GET_COMPUTED_STYLE: Get specific CSS property
   ├── GET_SHADOW_DOM: Traverse shadow DOM trees
   ├── GET_IFRAME_CONTENT: Access iframe content safely
   └── GET_ELEMENT_PATH: Get CSS selector path to element

   Implementation: ElementPropertyManager
   • Property validation and type coercion
   • Cache mechanism for frequently accessed properties
   • Shadow DOM traversal support
   • iframe and cross-frame access
   • Lines of code: 700-800 (with utilities)
   • Response time target: < 100ms (p95)

2. JavaScript Execution Context (7 commands)
   ├── EVAL_JAVASCRIPT: Evaluate expressions safely
   ├── CALL_FUNCTION: Call global functions
   ├── GET_GLOBAL_VARIABLE: Access window scope variables
   ├── SET_GLOBAL_VARIABLE: Set window scope variables
   ├── EXECUTE_WITH_CONTEXT: Run code in specific context
   ├── CALL_BROWSER_API: Call API methods via whitelist
   └── CHECK_JAVASCRIPT_SYNTAX: Validate JS syntax

   Implementation: JavaScriptContextManager + SandboxExecutor
   • Sandbox isolation for safe execution
   • API whitelist system
   • Timeout enforcement (30sec default)
   • Console output capture
   • Error tracking with stack traces
   • Lines of code: 900-1,000
   • Response time target: < 200ms (p95)

3. Network-Level Control (8 commands)
   ├── GET_PENDING_REQUESTS: List in-flight network requests
   ├── GET_REQUEST_DETAILS: Get request metadata
   ├── MODIFY_REQUEST: Modify request before sending
   ├── MOCK_REQUEST_RESPONSE: Mock response for request
   ├── GET_RESPONSE_BODY: Get response body
   ├── SET_REQUEST_INTERCEPTION: Set URL-based rules
   ├── GET_NETWORK_STATS: Get network statistics
   └── REPLAY_REQUEST: Replay a request with modifications

   Implementation: RequestInterceptionManager (extended)
   • Chrome DevTools Protocol integration
   • Request/response interception
   • Request modification and replay
   • Network statistics collection
   • Response caching for analysis
   • Lines of code: 500-600
   • Response time target: < 100ms (p95)

4. Storage Access (7 commands)
   ├── GET_LOCALSTORAGE: Access localStorage items
   ├── SET_LOCALSTORAGE: Write localStorage
   ├── DELETE_LOCALSTORAGE: Remove localStorage item
   ├── GET_SESSIONSTORAGE: Access sessionStorage
   ├── ACCESS_INDEXEDDB: Query IndexedDB stores
   ├── GET_INDEXEDDB_METADATA: Get database metadata
   └── ACCESS_CACHE_API: Access service worker cache

   Implementation: StorageAccessManager
   • localStorage/sessionStorage abstraction
   • IndexedDB traversal and querying
   • Cache API access
   • Pattern-based key filtering
   • Storage quota handling
   • Lines of code: 600-700
   • Response time target: < 100ms (p95)

5. DevTools Protocol Access (7 commands)
   ├── GET_DEVTOOLS_SESSION: Get session info
   ├── ENABLE_DEVTOOLS_DOMAIN: Enable debugger domain
   ├── GET_PERFORMANCE_METRICS: Get page metrics
   ├── SET_BREAKPOINT: Set JavaScript breakpoint
   ├── GET_MEMORY_INFO: Get heap memory info
   ├── GET_CPU_PROFILE: Get CPU profile
   └── START_PERFORMANCE_TRACE: Start performance trace

   Implementation: DevToolsManager
   • Chrome Remote Interface integration
   • Domain enablement and feature detection
   • Memory and CPU profiling
   • Performance tracing
   • Breakpoint management
   • Lines of code: 700-800
   • Response time target: < 200ms (p95)

AREA 2: CONTENT INJECTION & MODIFICATION (25 Commands)
═════════════════════════════════════════════════════════════════════════════

1. CSS Injection (8 commands)
   ├── INJECT_CSS: Inject CSS from string
   ├── INJECT_CSS_URL: Load CSS from URL
   ├── CREATE_STYLESHEET: Create dynamic stylesheet
   ├── MODIFY_STYLE_RULE: Edit specific CSS rule
   ├── REMOVE_INJECTED_CSS: Remove injected styles
   ├── APPLY_ANIMATION: Apply CSS animation
   ├── INJECT_THEME: Inject color theme
   └── GET_STYLESHEETS: List all stylesheets

   Implementation: CSSInjectionManager + CSSValidator
   • CSS syntax validation
   • Security checks for malicious CSS
   • Dynamic stylesheet management
   • Animation registration and tracking
   • Theme color variables
   • Lines of code: 700-800
   • Response time target: < 100ms (p95)

2. JavaScript Injection (8 commands)
   ├── INJECT_SCRIPT: Inject JS from string
   ├── INJECT_SCRIPT_URL: Load script from URL
   ├── EXECUTE_IMMEDIATE: Execute immediately
   ├── CREATE_PERSISTENT_SCRIPT: Create persistent script
   ├── REMOVE_INJECTED_SCRIPT: Remove script
   ├── MONKEY_PATCH: Replace function
   ├── HOOK_FUNCTION: Hook function calls
   └── GET_INJECTED_SCRIPTS: List injected scripts

   Implementation: JavaScriptInjectionManager
   • Script syntax validation
   • Monkey-patching framework
   • Function hooking with call capture
   • Execution log tracking
   • Persistent script support
   • Lines of code: 700-800
   • Response time target: < 150ms (p95)

3. DOM Manipulation (13 commands)
   ├── CREATE_ELEMENT: Create DOM element
   ├── CLONE_ELEMENT: Clone element (deep/shallow)
   ├── DELETE_ELEMENT: Remove element
   ├── MODIFY_ATTRIBUTES: Update attributes
   ├── SET_ELEMENT_CONTENT: Set content
   ├── APPEND_CHILD: Add child element
   ├── INSERT_BEFORE: Insert before reference
   ├── REPLACE_ELEMENT: Replace element
   ├── GET_ELEMENT_CHILDREN: Get children
   ├── BATCH_CREATE_ELEMENTS: Create multiple
   ├── BATCH_MODIFY_ATTRIBUTES: Modify multiple
   ├── BATCH_DELETE_ELEMENTS: Delete multiple
   └── WRAP_ELEMENT: Wrap in container

   Implementation: DOMManipulationManager + ElementRegistry
   • Element creation and lifecycle management
   • Batch operation optimization
   • Element registry for tracking
   • Operation history tracking
   • Rollback capability
   • Lines of code: 800-900
   • Response time target: < 100ms (p95)

## Technology Detection Module

### Overview

Technology Detection enables real-time identification of web frameworks, CMSs, servers, analytics platforms, and other technologies used in a website. With 50+ technology signatures and multiple detection methods, achieve 95%+ accuracy with sub-2-second response times.

**Perfect for:**
- Competitive technology intelligence
- Vulnerability assessment (knowing tech = knowing common CVEs)
- OSINT investigations
- Website profiling
- Threat intelligence gathering

### Getting Started (5 Minutes)

```javascript
// 1. Connect to Basset Hound WebSocket
const ws = new WebSocket('ws://localhost:8765');

// 2. Navigate to a website
ws.send(JSON.stringify({
  id: 1,
  command: 'navigate',
  url: 'https://example.com'
}));

// 3. Wait for page load, then detect technologies
setTimeout(() => {
  ws.send(JSON.stringify({
    id: 2,
    command: 'detect_technologies',
    includeVersions: true,
    confidence_threshold: 80
  }));
}, 3000);

// 4. Process results
ws.on('message', (data) => {
  const msg = JSON.parse(data);
  if (msg.id === 2 && msg.success) {
    console.log('Detected technologies:');
    msg.data.technologies.forEach(tech => {
      console.log(`${tech.name} (${tech.version || 'unknown'}) - ${tech.confidence}% confidence`);
    });
  }
});
```

### Complete Feature Guide

#### Detection Methods

Technology detection uses multiple methods to increase accuracy:

1. **HTTP Headers** - Server headers, X-Powered-By, X-Aspnet-Version, etc.
   - Fast, reliable for servers
   - Example: `Server: Apache/2.4.41 (Ubuntu)`

2. **HTML Patterns** - Meta tags, comments, path patterns
   - Catches CMS, frameworks, analytics tools
   - Example: `<meta name="generator" content="WordPress 6.2">`

3. **CSS/JavaScript Detection** - Framework-specific classes, attributes
   - Identifies frontend frameworks
   - Example: `data-react-root` attribute indicates React

4. **Cookies & Local Storage** - Third-party tracking cookies
   - Identifies analytics, ad networks, tracking services
   - Example: `_ga` cookie indicates Google Analytics

5. **DNS Records** - MX, CNAME records point to services
   - Email providers, CDN detection
   - Example: CNAME points to CloudFlare CDN

**Web Servers:**
- Apache, Nginx, IIS, LiteSpeed, Caddy, Node.js/Express

**CMS Platforms:**
- WordPress, Drupal, Joomla, Magento, Shopify, Ghost, Wix, Squarespace

### API Reference

#### Command: `detect_technologies`

**Description:** Analyze current page and detect installed technologies

**Request:**

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `includeVersions` | boolean | true | Include detected version numbers |
| `confidence_threshold` | number | 75 | Only return detections >= this confidence (0-100) |
| `detectionMethods` | array | all | Which detection methods to use |
| `cacheName` | string | null | Cache key for reuse (optional) |

### FAQ

**Q: How accurate is technology detection?**  
A: 95%+ for server technologies (HTTP headers). Frontend frameworks vary 85-95% depending on whether they expose patterns. Always review evidence before relying on results.

**Q: Can I detect technologies on pages that require login?**  
A: Yes! First use cookies/session commands to authenticate, then run detection. The technology detection happens on the DOM/headers you have access to.

**Q: Why does the same site show different technologies on different runs?**  
A: Sites may dynamically load different tech based on user agent, geolocation, or A/B testing. Run detection multiple times with different profiles for comprehensive results.

**Q: How do I integrate detected technologies with my OSINT workflow?**  
A: Use `export_to_platform` command to export to Shodan, Maltego, MISP, or other OSINT tools. See Platform Integrations section.

## Forensic Evidence Export

### Overview

Forensic Evidence Export enables creation of court-ready evidence packages with cryptographic verification and chain-of-custody documentation. Compliant with SWGDE guidelines and following ISO/IEC 27037 principles for handling digital evidence.

**Perfect for:**
- Legal investigations
- Law enforcement operations
- Incident response
- Digital forensics reports
- Admissible evidence compilation

### Getting Started (5 Minutes)

```javascript
// 1. Initialize forensic evidence system
ws.send(JSON.stringify({
  id: 1,
  command: 'init_evidence_chain',
  basePath: '/evidence/case-2026-12345',
  autoVerify: true,
  autoSeal: true
}));

// 2. Create investigation
ws.send(JSON.stringify({
  id: 2,
  command: 'create_investigation',
  name: 'Malware Analysis - Case ABC',
  description: 'Investigation of malicious website',
  investigator: 'John Doe'
}));

// 3. Collect evidence during investigation
// (automatically records all navigation, screenshots, extractions)

// 4. Export as forensic package
ws.send(JSON.stringify({
  id: 3,
  command: 'export_forensic_evidence',
  investigationId: '...',
  format: 'zip',
  includeReport: true,
  includeChainOfCustody: true,
  signature: 'john.doe@agency.gov'
}));

// 5. Receive ZIP with all evidence and documentation
```

### Complete Feature Guide

#### Evidence Types Tracked

1. **Screenshots** - Page captures at key moments
   - Full page, viewport, element, and area captures
   - Timestamp and session ID recorded
   - PNG or JPEG format with metadata

2. **Network Traffic** - HTTP requests, responses, WebSocket messages
   - Full request/response bodies
   - Timing information
   - Certificate validation logs

3. **DOM State** - HTML content at different points
   - Extraction of page structure
   - Form detection and content
   - Dynamic content tracking

4. **Console Output** - JavaScript execution results
   - Script outputs
   - Errors and warnings
   - Network timing logs

5. **Metadata** - Information about the investigation
   - URLs visited
   - Actions taken (clicks, fills, navigation)
   - Session information
   - Timing and sequence

#### Chain of Custody Features

- **Actor Recording** - Who performed what action (investigator ID, timestamp)
- **Integrity Hashing** - SHA-256 hashes of all evidence
- **Cryptographic Manifest** - Signed list of all evidence
- **Immutability** - Once sealed, evidence cannot be modified
- **Audit Trail** - Complete log of who accessed what, when
- **Legal Compliance** - Follows SWGDE and ISO/IEC 27037 guidelines

### FAQ

**Q: Is this evidence admissible in court?**  
A: Properly collected and documented evidence from Basset Hound can be admissible if proper chain of custody is maintained. Consult with legal counsel about jurisdiction-specific requirements.

**Q: How long is evidence retained?**  
A: Exported evidence packages have a default retention of 90 days. Extend via parameters. Original evidence in the browser is cleared when sessions are destroyed.

**Q: Can I export evidence multiple times?**  
A: Yes, each export creates a new sealed copy with different hash. All exports are tracked in the chain of custody log.

**Q: What format should I use for court submissions?**  
A: ZIP is most common and portable. ISO format provides write-protection and forensic-grade preservation. Consult with your legal team.

## Platform Integrations

### Overview

Platform Integrations connect Basset Hound to the broader OSINT ecosystem, enabling seamless data export to industry-standard tools like Shodan, Maltego, MISP, and others. Each platform has optimized export formats and webhook support for real-time notifications.

**Perfect for:**
- OSINT workflow automation
- Threat intelligence correlation
- Multi-tool investigations
- Real-time alerting systems
- Incident response automation

3. **MISP** - Malware Information Sharing Platform

5. **STIX/TAXII** - Standardized threat intelligence format

### FAQ

**Q: Do I need accounts on all platforms?**  
A: No. Configure only the platforms you use. Basset Hound supports 5+ integrations, but you only need to setup the ones relevant to your workflow.

**Q: Can I export to multiple platforms at once?**  
A: Yes! Call `export_to_platform` multiple times for different platforms, or use the batch export feature to export to all configured platforms simultaneously.

**Q: Are webhooks required?**  
A: No, webhooks are optional. Without webhooks, you poll for export status. With webhooks, you get instant notifications when exports complete.

## Advanced JavaScript Execution

### Overview

Advanced JavaScript Execution enables running complex custom scripts within the browser context with sandboxing, timeout protection, and comprehensive error handling. Perfect for extracting data from complex Single-Page Applications (SPAs) with React, Vue, Angular, and other frameworks.

**Perfect for:**
- SPA data extraction (React, Vue, Angular)
- Custom payload deployment
- Complex DOM manipulation
- State extraction from JavaScript frameworks
- Template-based extraction

### Getting Started (5 Minutes)

```javascript
// 1. Simple script execution
ws.send(JSON.stringify({
  id: 1,
  command: 'execute_script',
  script: `
    // Returns data to Basset Hound
    return {
      pageTitle: document.title,
      bodyText: document.body.innerText.substring(0, 1000),
      links: Array.from(document.querySelectorAll('a'))
        .map(a => ({ text: a.textContent, href: a.href }))
    };
  `,
  timeout: 5000
}));

// 2. Using templates for common patterns
ws.send(JSON.stringify({
  id: 2,
  command: 'execute_script_template',
  template: 'react-state-extractor',
  componentName: 'MainComponent'
}));

// 3. Process results
ws.on('message', (data) => {
  const msg = JSON.parse(data);
  if (msg.id === 1 && msg.success) {
    console.log('Extracted data:', msg.data.result);
  }
});
```

```javascript
// Deploy custom payload in sandboxed execution
ws.send(JSON.stringify({
  id: 1,
  command: 'execute_script',
  script: `
    // Sandboxed execution - cannot escape or damage host
    const results = [];
    
    // Monitor for network requests
    const originalFetch = window.fetch;
    window.fetch = function(...args) {
      results.push({
        type: 'fetch',
        url: args[0],
        method: args[1]?.method || 'GET',
        timestamp: new Date().toISOString()
      });
      return originalFetch.apply(this, args);
    };
    
    // Trigger requests by clicking elements
    document.querySelectorAll('button').forEach(btn => {
      if (btn.textContent.includes('Submit')) {
        btn.click();
      }
    });
    
    return { requests: results };
  `,
  timeout: 5000,
  captureConsole: true
}));
```

### FAQ

**Q: Can executed scripts break out of the sandbox?**  
A: No. Scripts execute in a v8 isolate with restricted access. They cannot modify the Basset Hound process or filesystem.

**Q: What timeout should I use?**  
A: Default 30 seconds is safe for most scripts. Use 5000ms for quick extractions, up to 60000ms for complex operations.

**Q: Can I use async/await in scripts?**  
A: Yes! Scripts support async/await, promises, and all modern JavaScript.

**Q: How do I debug failing scripts?**  
A: Check the `console` and `errors` fields in the response. Set `captureConsole: true` to see console.log output.

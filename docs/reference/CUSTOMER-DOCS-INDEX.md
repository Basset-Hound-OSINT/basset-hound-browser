## Writing Guidelines (for adding more docs)

### Style
- Write for customers, not engineers
- Use simple language (avoid jargon)
- Include examples and screenshots
- Provide step-by-step for complex tasks
- Use tables for comparisons

### Structure
- Start with problem/goal
- Explain the "why"
- Provide step-by-step solution
- End with verification/result
- Add troubleshooting at end

### Formatting
- Use markdown
- Headings: H1 (docs), H2 (sections), H3 (subsections)
- Code blocks with language: ```bash, ```json, ```javascript
- Lists for steps or options
- Tables for comparisons
- Blockquotes for tips/warnings

### Examples
- Copy-paste ready code
- Real product names (not "example.com")
- Common use cases
- Screenshots referenced: [Screenshot: Dashboard with alerts]
